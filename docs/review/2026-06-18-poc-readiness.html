3 PoC 最小关键路径

两条线泳道图：线 A — CLI 版 / 线 B — GUI 弹窗版。菱形 ◆ 标外部阻塞点。

图 1 · 双线泳道关键路径：线 A（CLI）7 步、线 B（GUI）5 步叠加；黄色菱形为外部依赖阻塞点。

线 A — CLI 版可演示 PoC（除真 key 无硬阻塞）

  1. commit / push 全部修复
  2. cargo build ＋ sieve setup 一键注入
  3. CLI 真启动 smoke
  4. 【需用户】拿 2 把 key ＋ SIEVE_NO_UPDATE 启 daemon
  5. 真流量：粘 .env 看脱敏（OUT-01）＋触发 signTransaction / curl|sh 看拦截（IN-CR-05 / 02）
  6. 查 audit.db 的 DecisionMade / OutboundRedacted 行
  7. tcpdump 确认 path prefix

总账：我侧 1–2h；外部阻塞＝用户给 2 把 key；给了当天出第一次 demo。

线 B — GUI 弹窗完整版（叠加，需 Xcode）

  1. 【需用户】Xcode 就位
  2. xcodebuild 编译 GUI
  3. 生成 Sparkle 密钥并 inject
  4. 清数据完整性债（list_rules / wait_for_ipc / direction）
  5. 真 daemon ↔ GUI 联调（菜单栏五态 → BIP39 触发 → HIPS 弹窗倒计时 → 拒绝 → 红图标 + History + 导出 CSV）

总账：Xcode 就位后我侧约半天。  粗估总账：CLI demo＝用户给 key 后 0.5 天内；GUI 完整＝Xcode 就位后 +1 天内。

4 明确砍掉的（GA 才要）

以下全部移出 PoC 范围，GA 阶段才纳入。

Linux / Windows 多平台 + 拦截引擎 trait 非 macOS 实现 网络隔离 enforcement（ADR-027 pf/nftables） 行为序列升 Block（需 4 周 ≥50 样本） 运维服务端全栈（TODO-13~16：域名 / GCP KMS / CF Workers+D1 / CDN） 签名分发链（sigstore / notarization / appcast / .dmg） Stripe + 海外主体 闭测招募

5 风险 / 未知

等级：🔴 高 🟡 中 🟢 低

等级风险说明 → 缓解
🔴 真 Claude Code 行为未知 全验证基于 mock，真 API 的 SSE 分块 / tool_use 结构可能与 fixture 有微差 → §3.3 必须真跑。
🔴 GUI 能否在机上构建 + 签名 SwiftUI 层从未 xcodebuild 编译，开发机无 Xcode → 装 Xcode 后第一件事就 BUILD，失败则 CLI 版先顶。
🟡 真 API key 可得性 外部依赖，卡用户侧。
🟡 真 dogfood FP 率未知 当前 FP 0% 是 fuzzing 模拟数据，真合约代码可能误报 → demo 前用真 .env + 真合约预跑。
🟢 提交 / push 后 dogfood 是否仍全绿 跨仓 fixture 漂移有复发风险 → push 后立即重跑 dogfood.sh。
