# ADR-029: 装机量优先,延后商业化 + 拒绝中转站认证收费模式

## 状态
**Accepted**
> 决策日期:2026-05-05
> 范围:商业化路径选择 + 已排除方向
> 关联:[ADR-005](./ADR-005-overseas-legal-entity.md)(海外主体收款) / [ADR-011](./ADR-011-private-until-ga.md)(GA 前私有)

## 背景

GA 前一个月需要明确商业化路径。`doskey` 提出过「向 token 中转站收认证费 + 在导流页给广告位」的方案:

- 普通用户(自己安装)免费
- 在积累足够装机量后,向中转站收 ~500USD/月「Sieve 认证」费,用 Sieve 自身流量审计能力为该中转站背书「全部流量经过 Sieve 不被污染」
- 做一个公开的「认证中转站索引页」,中转站可购买更高排名 / 投放广告

该方案与 PRD §11 现有商业化策略(订阅 + 加密支付双通道)正交,需独立判定。

## 决策

**1. Phase 1(GA 前 6–12 个月)走纯免费模式,前唯一目标是装机量。**

- 普通用户安装、规则更新、所有检测能力全部免费,无任何付费门槛
- 不做免费试用 / freemium 限速 / 用量上限
- 不接入任何 B 端付费功能(企业版 / 团队版 / 审计仪表盘)
- 该阶段不做收入预测与 MRR KPI;唯一对外指标是装机量与日活(数据来源见 [ADR-030](./ADR-030-update-telemetry-channel.md))

**2. 拒绝「中转站认证 + 排名广告」收费模式,永久排除。**

直接原因是评级机构的利益冲突——同一主体既颁发「中立认证」又收取被认证方的广告费,会摧毁认证本身的可信度(参考 Moody's 2008 / App Store ranking 争议)。Sieve 的产品定位本身建立在「客观流量审计」之上,一旦有付费排名,用户会直接质疑「Sieve 是否还在如实审计付费中转站」。

**3. 未来商业化的预备方向(Phase 2+,非当前优先级):**

仅作记录,不在 GA 前决策:

| 方向 | 类型 | 说明 |
|------|------|------|
| 用户端 Pro 订阅 | C 端 | 多账号聚合 / 高级用量分析 / 团队共享配置(工具型功能) |
| 中转站 B 端订阅 | B 端 | 中转站买实时监控 / 异常告警 / 用量报表(工具型,**非结论型**) |
| 客观技术认证 | 中立服务 | 若做认证业务,**认证与广告必须严格分离**:认证按公开客观标准 pass/fail,只收审计成本费;广告位明确标「赞助」且不影响认证结论 |

**4. 商业化前置触发条件:**

任何 B 端商业化讨论必须先满足装机量阈值(具体阈值待 6 个月装机数据出来后定)。在装机量未到阈值前,不接收来自中转站的合作 inbound,统一回复「Sieve 当前仅向用户提供本地审计工具,不做中转站合作业务」。

## 影响

### 正面影响
- 与产品定位(客观本地审计)完全一致,不需要话术调和
- GA 前避开商业化讨论的精力消耗,聚焦工程
- 装机量数据本身就是后续任何变现路径的最大筹码
- 隐私话术清爽(免费 + 不上传数据 + 可选关闭遥测,见 [ADR-030](./ADR-030-update-telemetry-channel.md))

### 负面影响
- GA 后 6–12 个月零收入,完全依赖 `doskey` 自有资金支撑
- 与 [ADR-005](./ADR-005-overseas-legal-entity.md) 的海外公司收款主体延期使用,Stripe / 加密支付通道的实际启用推后到 Phase 2
- 装机量数据若长期不达预期,需要回到本 ADR 重新评估「免费优先」是否还成立(不把这种回退视为失败,而是数据驱动决策)
