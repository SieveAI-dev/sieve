# ADR-005: 海外公司主体作为收款与营销载体

## 状态

**已接受**（v1.3 锁定执行）

> 决策日期：2026-04-26
> 范围：Sieve 商业化主体、收款渠道、营销载体
> 关联 PRD：[v1.3 §1.4、§11.5](../prd/sieve-prd-v1.3.md)

---

## 背景

doskey 在中国大陆做研发。Sieve 的目标 ARR 是 $300K（18 个月）→ $700K+（24 个月稳态），P0 用户是"crypto-native AI 重度开发者"，地理分布以海外为主（PRD §3.1）。商业化必须解决两件事：

1. **收款主体**：Stripe / Crypto 支付通道需要合法的公司或个体经营主体；
2. **营销载体**：在中国大陆"虚拟货币业务、信息中介、定价、导流、互联网展示"被持续高压监管（PRD §11.5.2），doskey 个人在国内的营销动作有约束。

**v1.2 的设计漏洞**：完全没考虑中国大陆合规边界，把"法律实体"标为"待定"。这是 GPT-5.5 review #1 抛出的硬卡点。如果不锁定主体形态，Stripe 接入（Week 7-8）会被卡住，整个 12 周里程碑会延期。

候选主体：

| 主体形态 | 注册周期 | 银行账户对中国国籍 | Stripe 集成 | 后续家人移居弹性 | 税务复杂度 |
|---------|---------|------------------|------------|----------------|----------|
| 中国大陆个人/个体户 | 1–2 周 | ✅ | ❌ Stripe 不支持 | 无 | 低（个人所得税） |
| 中国大陆有限公司 | 4–6 周 | ✅ | ❌ Stripe 不支持 | 无 | 中 |
| **香港有限公司**（Ltd.） | 4–6 周（可加急 2 周） | ✅ 友好 | ✅ Stripe Asia 友好 | ✅ 香港身份转 | 中（利得税 16.5%，离岸收入豁免） |
| 新加坡 Pte Ltd | 6–10 周 | ⚠️ 需本地董事 | ✅ | ✅ 新加坡身份转 | 中-高（CIT 17%，但有税务优惠） |
| 美国 Stripe Atlas LLC | 2 周 | ❌ 银行账户难开 | ✅✅ 最丝滑 | 中 | 高（联邦 + 州 + 1099/W-9 复杂） |

**Stripe 在中国大陆不接受个人/个体户作为收款主体**——这一条直接淘汰了所有大陆主体。**BVI / Cayman 等离岸主体明确不在候选范围**——Stripe 不友好且 PRD §1.4 未授权该路径。

## 决策

### 1. 主体选择优先级

**首选：香港有限公司**

理由：

- 注册周期 4–6 周，可在 Week 1 启动、Week 7-8 之前拿到执照赶上 Stripe 接入（PRD §10.1 Week 7-8）；
- 银行账户对中国国籍**最友好**（恒生 / HSBC / 众安等都接受大陆居民开户）；
- 与 Stripe Asia 集成丝滑，可直连港币 / 美元收款；
- 利得税 16.5%，**离岸收入可申请豁免**（Sieve 用户在海外，符合离岸条件）；
- 香港身份的家人移居弹性最大（如 doskey 家庭未来移居香港，业务可平滑迁移）；
- 物理上 doskey 可短时间出境办理。

**次选：新加坡 Pte Ltd**

适用条件：如果香港注册因任何原因受阻。

代价：

- 注册周期 6–10 周（本地 nominee director + 实际办公地址要求繁琐）；
- 银行账户开立较难（需要 KYC 加 nominee 配合）；
- Phase 1 内可能无法完成。

**第三：美国 Stripe Atlas（Delaware LLC）**

适用条件：前两项都不可行。

代价：

- Stripe 集成最丝滑（Atlas 是 Stripe 自家产品）；
- **银行账户对中国国籍极不友好**（Mercury / Brex / Relay 都做了背景审查收紧）；
- 美国税法复杂度极高（联邦 + 州 + 1099/W-9 + IRS Schedule C）；
- 一人公司 Phase 1 没有这个税务复杂度处理能力。

| 加密支付通道集成 | ⭐⭐⭐⭐（Coinbase Commerce / Stripe Crypto） | ⭐⭐⭐⭐ | ⭐⭐⭐⭐⭐ |
| 后续家人移居弹性 | ⭐⭐⭐⭐⭐（地理就近） | ⭐⭐⭐⭐ | ⭐⭐ |

| **加权评分** | **首选** | 次选 | 第三 |

### 3. 加密支付双通道

PRD §11.5.1 要求**收款渠道双通道**：

- **主通道：Stripe**——绑定海外公司后启用；订阅式 $49/月（年付 $490 9 折）；
- **备选通道：USDC / USDT 收款**——服务于 crypto-native 用户的偏好（部分用户根本不愿用信用卡付加密产品）。

加密支付实现路径（Open Question，§14 第 5 条）：

| 方案 | 优势 | 代价 |
|------|------|------|
| **Stripe Crypto Onramp** | 与 Stripe 同栈、合规省心 | 不是真正的链上收款，是法币 → 加密代理 |
| **Coinbase Commerce** | 真正链上收款（USDC on Ethereum / Polygon / Solana） | 需独立 webhook + 订阅状态机 |
| **自部署链上签名校验** | 完全自主、无中间方 | 工程复杂度高，Phase 1 不做 |

**Phase 1 采纳**：Stripe（法币）+ Coinbase Commerce（链上 USDC）双通道。

### 4. 营销渠道分级

直接对齐 PRD §11.5.2 表格：

| 渠道 | 风险等级 | 内容策略 |
|------|---------|---------|
| Twitter（英文为主） | 低 | **首发主战场**——产品营销 + 技术研究两类内容均可 |
| Hacker News / Reddit r/ethdev / r/cryptocurrency | 低 | 文章 1 + 2 + 3 同步发，英文 |
| Mirror / Substack 英文 blog | 低 | 长文主战场 |
| GitHub README | 低 | 英文为主，中文翻译次之 |
| 即刻 | **中** | **只发 AI 安全研究角度**，不直接发 crypto 资产防护叙事，不附 landing page 链接 |
| V2EX | **中** | 同即刻 |
| 微信公众号 | **高** | 不在主要营销规划 |
| 小红书 / 知乎 / B 站 | **高** | 不规划 |
| 国内 Crypto 社群（Telegram 中文群除外） | **极高** | 不主动营销 |

**核心原则**（PRD §11.5.2）：**境内渠道发研究内容，境外渠道发产品营销**。doskey 个人在国内的存在不要把"卖 crypto 工具"和"做安全研究"两件事混在一起公开讲。

## 影响

### 正面影响

1. **解锁 Stripe**：海外公司主体让收款合法合规，Phase 1 12 周里程碑不被收款主体卡住；
2. **合规边界清晰**：海外主体 + 境内境外渠道分级 → 直接降低 doskey 个人在国内的法律风险；
3. **未来弹性**：香港主体可平滑迁移到新加坡 / 美国；如果 doskey 家庭移居海外，业务无需重写；
4. **Crypto 用户友好**：双通道（Stripe + USDC）覆盖法币和加密圈两类付费偏好；
5. **税务可优化**：香港离岸豁免对一人公司是合理的税务结构。

### 负面影响

1. **注册周期占用 4–6 周**：Week 1 必须启动；如果 Week 5 还没拿到执照，Stripe 接入会延期 → 风险登记册第 12 条已记录；
2. **注册成本**：香港有限公司初次注册 + 秘书公司 + 银行账户开立 + 年审，**首年约 HKD 15K–25K**；后续年度约 HKD 8K–15K；
3. **海外公司维护负担**：年审、报税、利得税申报需要找秘书公司协助（建议直接付费 outsource 给如 OneIBC / Bridges 之类的服务商）；
4. **境内营销受限**：不能用微信公众号 / 小红书 / B 站等大流量国内渠道——这是合规换来的代价；
5. **跨境资金流动**：海外公司利润分红回大陆个人账户需要走合规通道（如香港优才 / 投资移民身份变更后才方便），Phase 1 内利润留存海外即可。
