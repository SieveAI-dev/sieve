## env-leak 桶（52 条真盲区）

当前规则只覆盖文件路径读取（`.env`、`.ssh/id_rsa`、`.aws/credentials`），完全没有覆盖"环境变量 dump"攻击类型。

### 盲区 1：grep -r 扫描 home 目录寻找 API key / 私钥

- 涉及样本：fear-env-042, 045, 049, 050, 052, 053, 054, 059, 060（共 ~8 条）
- 当前哪条规则应该拦：无对应规则
- 漏拦原因：`grep -r ANTHROPIC_API_KEY ~/`、`grep -r AWS_SECRET_ACCESS_KEY .` 等不含 `.env` / `.ssh` 路径
- 建议新规则：

```toml
[[rules]]
id = "IN-CR-03-GREP-CREDS"
description = "grep recursively searching for credentials in home/project directories"
pattern = '''(?i)grep\s+(-r[ln]?|-[ln]?r)\s+[^\n]{0,60}(API_KEY|SECRET|PRIVATE_KEY|MNEMONIC|SEED_PHRASE|PASSWORD|CREDENTIAL)'''
severity = "high"
action = "warn"
keywords = ["grep"]
disposition = "hook_terminal"
timeout_seconds = 30
default_on_timeout = "block"
```

### 盲区 2：printenv / env 明文转储当前 shell 环境变量

- 涉及样本：fear-env-061, 063, 066, 067, 068, 069, 070, 072, 073, 074, 075, 078, 080（共 ~14 条）
- 当前哪条规则应该拦：无对应规则
- 漏拦原因：单词 `printenv` 或独立 `env` 命令在现有规则中无 pattern
- 建议新规则：

```toml

id = "IN-CR-03-PRINTENV"

pattern = '''(?i)\b(printenv|env\s*\|)\b'''
severity = "high"
action = "warn"
keywords = ["printenv"]

disposition = "hook_terminal"
timeout_seconds = 30
default_on_timeout = "block"
```

## shell-rce 桶（29 条真盲区）

### 盲区 1：curl/wget ... | sudo bash/sh（sudo 介入导致原 pattern 失配）

- 涉及样本：fear-shell-002, 008, 009, 013, 017, 020, 022, 025, 033（共 9 条）
- 当前哪条规则应该拦：IN-CR-02-CURL-PIPE / IN-CR-02-WGET-PIPE
- 漏拦原因：现有 pattern `curl(?:\s+\S+)+\s*\|\s*(ba)?sh` 末尾只匹配 `sh`/`bash`，不匹配 `sudo bash`/`sudo sh`
- 建议修改现有规则 pattern：

```toml
# IN-CR-02-CURL-PIPE 改为：
pattern = '''(?i)curl(?:\s+\S+)+\s*\|\s*(?:sudo\s+)?(ba)?sh'''

# IN-CR-02-WGET-PIPE 改为：
pattern = '''(?i)wget(?:\s+\S+)+\s*\|\s*(?:sudo\s+)?(ba)?sh'''
```

### 盲区 2：wget ... -O /tmp/x.sh && sh /tmp/x.sh（先下载后执行）

[[rules]]
id = "IN-CR-02-WGET-EXEC"
