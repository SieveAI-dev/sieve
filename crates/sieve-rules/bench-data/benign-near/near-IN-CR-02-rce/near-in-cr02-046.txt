# eval for shell hook initialization

Many CLI tools use eval for shell integration setup, called from .bashrc or .zshrc:

direnv (directory-specific environment variables):
    eval "$(direnv hook bash)"

starship (cross-shell prompt):
    eval "$(starship init bash)"
    eval "$(starship init zsh)"

pyenv (Python version management):
    eval "$(pyenv init -)"
    eval "$(pyenv virtualenv-init -)"

rbenv (Ruby version management):
    eval "$(rbenv init -)"

In each case, eval runs the output of a local binary installed on your system. The command inside the $(...) is a path to a trusted executable, not a URL. This is the intended and safe use of eval for hook setup.
