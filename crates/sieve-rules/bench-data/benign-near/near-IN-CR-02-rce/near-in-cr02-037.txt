# Bash pitfalls: pipe-to-shell supply chain risk

From the "Bash Pitfalls" wiki (adapted):

Running arbitrary code from the internet is a significant risk. The "install via pipe to shell" antipattern:

    # BAD PATTERN (do not use):
    # wget -O - https://example.com/install | sh

has been exploited in supply chain attacks where:
- CDN compromise served malicious scripts
- DNS hijacking redirected legitimate domains
- Man-in-the-middle inserted additional commands

Safer alternatives:
1. Use a package manager (apt, brew, cargo install, npm install)
2. Download → inspect → run (see previous note)
3. Use cosign / sigstore to verify the binary signature before execution

The goal is breaking the unbroken chain from network to shell execution.
