# wget vs curl: which to use when?

Question from Stack Overflow (adapted):

> I want to download a binary release from GitHub and use it in a CI pipeline.
> Should I use wget or curl?

Answer: either works. A common idiom is:

    # curl variant (writes to stdout, redirect to file)
    curl -sL https://github.com/org/tool/releases/download/v1.0/tool-linux-amd64 \
         -o /usr/local/bin/tool && chmod +x /usr/local/bin/tool

    # wget variant (saves to file directly)
    wget -q https://github.com/org/tool/releases/download/v1.0/tool-linux-amd64 \
         -O /usr/local/bin/tool && chmod +x /usr/local/bin/tool

Both download a pre-built binary. The key step is the chmod +x afterward. Verify the binary's SHA256 hash before making it executable in production pipelines.
