Turbopack and Vercel cache directories in a monorepo need periodic cleanup:

    rm -rf .turbo .vercel

.turbo/ is created by Turborepo and stores task output hashes and cached build results. .vercel/ is created by the Vercel CLI when you link a project locally. Both are gitignored by default.

In a Turborepo workspace with multiple packages, caches live under:

    .turbo/
    packages/web/.turbo/
    packages/api/.turbo/

To clean all turbo caches recursively:

    find . -type d -name .turbo -exec rm -rf {} +

Or use the built-in turbo command:

    turbo clean

After cleaning, the next `turbo build` will be a full cold build. Useful when cache invalidation bugs cause stale outputs to be served.
