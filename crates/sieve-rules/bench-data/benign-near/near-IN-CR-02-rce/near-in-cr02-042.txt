# Using HTTPS and checksum verification for wget downloads

Never download and run in a single step. Instead:

```bash
#!/usr/bin/env bash
set -euo pipefail

TOOL_URL="https://releases.example.com/tool-v1.2.3-linux-amd64.tar.gz"
EXPECTED_SHA256="a1b2c3d4e5f6..."  # from the releases page

# Step 1: download
wget -q "$TOOL_URL" -O /tmp/tool.tar.gz

# Step 2: verify
actual=$(sha256sum /tmp/tool.tar.gz | awk '{print $1}')
if [ "$actual" != "$EXPECTED_SHA256" ]; then
    echo "Checksum mismatch! Expected $EXPECTED_SHA256, got $actual" >&2
    rm -f /tmp/tool.tar.gz
    exit 1
fi

# Step 3: extract
tar -xzf /tmp/tool.tar.gz -C /usr/local/bin/
```

This pattern—download, verify, extract—is significantly safer than any pipe-to-shell variant.
