# dd: generate random bytes for cryptographic use

Generate 32 random bytes and encode as hex (for a secret key):

    dd bs=32 count=1 if=/dev/urandom 2>/dev/null | xxd -p -c 256

Output example:
    a3f8c2d1e5b490276f3c81da20e4b7f9c6a1d8e25b7f4309c1d6e8f2a

On modern systems, prefer openssl rand or Python secrets for cryptographic key generation—they handle OS-level randomness more explicitly:
    openssl rand -hex 32
    python3 -c "import secrets; print(secrets.token_hex(32))"

The dd approach works but reading exactly N bytes from /dev/urandom requires care with the bs/count parameters.
