Rust cargo build artifacts can grow very large. Selectively cleaning only debug artifacts:

    rm -rf target/debug

This removes the debug build directory but keeps target/release intact, so your release binary stays available. The full target/ tree can reach 10–30 GB on large projects; target/debug is usually the biggest chunk.

Alternative: use cargo-clean selectively:

    cargo clean --profile dev

But rm -rf target/debug is faster because cargo clean re-reads Cargo.lock and does extra work. If you just want disk space back quickly, the rm is fine.

To clean everything including release:

    cargo clean

Or equivalently:

    rm -rf target/

Both are safe—cargo will rebuild everything on next `cargo build`. The source code in src/ is untouched.
