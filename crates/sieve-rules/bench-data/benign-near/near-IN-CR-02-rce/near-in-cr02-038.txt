# Security review checklist for install scripts

When you encounter an install script from the internet, check for:

[ ] Does it use `curl ... | sh` or `wget ... | sh` internally? (red flag)
[ ] Does it call `eval` with externally-sourced data? (red flag)
[ ] Does it write to /etc/cron.d or LaunchAgents? (deserves scrutiny)
[ ] Does it modify PATH in shell rc files without asking?
[ ] Does it download additional binaries and execute them?
[ ] Is there a cosign / GPG signature for the released artifact?

Legitimate install scripts usually:
- Explain what they're doing with comments
- Ask for sudo only when strictly needed
- Support a --dry-run flag
- Document what changes they make to the system

If the script fails multiple checks above, use a manual installation method instead.
