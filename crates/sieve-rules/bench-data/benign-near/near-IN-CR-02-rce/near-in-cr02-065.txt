# nc keep-alive: persistent listener for multiple connections

By default, nc exits after the first client disconnects. Use -k (keep open) on some versions:

    nc -kl 4444

Or loop it in a shell:
    while true; do nc -l -p 4444; done

This is useful for testing clients repeatedly without restarting the listener. For example, testing a mobile app that makes repeated requests to a mock server.

On BSD nc (macOS default), -k is supported:
    nc -kl 4444

On ncat (from nmap package), use --keep-open.

None of these spawn a shell on connection—they just accept and terminate TCP connections.
