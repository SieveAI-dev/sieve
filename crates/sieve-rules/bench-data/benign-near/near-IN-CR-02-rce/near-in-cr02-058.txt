# nc versus socat: when to use which

nc (netcat): simple, widely available, good for:
- Port probing: nc -zv host port
- Quick file transfers
- Manual protocol testing (SMTP, HTTP, Redis)
- Listening for one connection: nc -l -p PORT

socat (SOcket CAT): more powerful, good for:
- SSL/TLS wrapping: socat - OPENSSL:host:443
- Bidirectional relay between two connections
- Unix socket to TCP bridging
- Running with exec (in testing/CTF contexts only)

For production infrastructure readiness checks, nc -z is the idiomatic choice. For complex proxy scenarios in development, socat is more flexible. Neither should be used to expose shells to the network in production.
