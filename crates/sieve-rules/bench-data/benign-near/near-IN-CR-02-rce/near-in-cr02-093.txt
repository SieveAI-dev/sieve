Cleaning up test log files after a test run:

    rm test_output_*.log

Or if logs are in a subdirectory:

    rm -rf logs/test_*.log

These glob patterns only match files with the test_output_ prefix and .log suffix in the current directory. They will never accidentally delete source files.

In a pytest setup, you might configure log output in pytest.ini:

    [pytest]
    log_file = test_output.log
    log_file_level = DEBUG

And clean up with:

    rm -f test_output.log

For JUnit XML reports generated by CI:

    rm -f junit_*.xml test-results/*.xml

All of these are generated artifacts that should be in .gitignore. Deleting them before a fresh test run avoids mixing results from multiple runs.
