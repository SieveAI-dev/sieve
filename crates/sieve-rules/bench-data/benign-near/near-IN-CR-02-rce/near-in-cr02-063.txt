# nc 用于检测防火墙端口是否开放

在 DevOps 场景中，快速检查某台机器的端口是否可达：

    nc -zv 10.0.1.50 443
    nc -zv 10.0.1.50 5432   # PostgreSQL
    nc -zv 10.0.1.50 6379   # Redis

-z：只做 TCP 握手，不发送数据
-v：打印连接结果

批量检查多个端口：
    for port in 80 443 8080 8443; do
        nc -zv myhost.internal $port 2>&1
    done

这是运维日常诊断工具，等价于 `telnet host port` 但不需要安装 telnet。
连接目标是普通 TCP 端口，不会在远端执行任何命令。
