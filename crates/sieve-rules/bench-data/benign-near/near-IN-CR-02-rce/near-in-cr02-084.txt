# rm -rf build artifacts in Makefile

```makefile
.PHONY: clean distclean

clean:
	rm -rf target/debug target/release
	rm -rf .next dist build
	rm -rf __pycache__ .pytest_cache .mypy_cache

distclean: clean
	rm -rf node_modules
	rm -rf .venv venv env

rebuild: clean
	cargo build --release
```

The rm -rf targets are all well-defined build output directories. The pattern `rm -rf <specific-directory>` is standard Makefile hygiene. Compare with the dangerous pattern `rm -rf /` which has no legitimate use.
