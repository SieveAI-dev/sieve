# curl vs wget: key differences

Both tools download files from URLs, but they differ:

curl:
- Better for API calls (rich header/body control)
- Supports many protocols: HTTP, HTTPS, FTP, SFTP, SCP, LDAP
- Does NOT follow redirects by default (add -L)
- Outputs to stdout by default

wget:
- Better for recursive website mirroring
- Follows redirects automatically
- Can continue interrupted downloads with -c
- Saves to file by default (not stdout)

For a one-off API test: prefer curl.
For downloading a directory tree: prefer wget.
For scripts that need to work on both macOS and Linux: check if both are installed.
