# wget vs curl security comparison

From a security standpoint, both wget and curl have the same fundamental risks:

- Both can download arbitrary content from the internet
- Both support HTTP, HTTPS, FTP
- Neither verifies content integrity by default
- Both can be used in dangerous pipe-to-shell patterns (DO NOT USE)

Neither tool is inherently more secure than the other. Security comes from how you use them:

UNSAFE (both tools):
  # wget variant — DO NOT RUN
  # wget -O - https://example.com/setup | sh

  # curl variant — DO NOT RUN
  # curl https://example.com/setup | sh

SAFE (both tools):
  curl -O https://example.com/setup.sh && sha256sum setup.sh && bash setup.sh
  wget https://example.com/setup.sh && sha256sum setup.sh && bash setup.sh
