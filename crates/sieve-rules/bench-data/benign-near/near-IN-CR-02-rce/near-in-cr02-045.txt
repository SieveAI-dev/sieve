# eval in legitimate shell usage: SSH agent

The canonical way to start ssh-agent and configure your shell:

    eval $(ssh-agent -s)

What this does:
- `ssh-agent -s` starts the agent and prints shell commands to stdout, like:
      SSH_AUTH_SOCK=/tmp/ssh-XXXXXX/agent.1234; export SSH_AUTH_SOCK;
      SSH_AGENT_PID=1234; export SSH_AGENT_PID;
- `eval` executes those variable assignments in the current shell

Without eval, you'd have to parse and source the output manually. This is a well-established shell idiom.

After eval, add your key:
    ssh-add ~/.ssh/id_ed25519

The input to eval here comes from a trusted local binary, not from the network.
