# How attackers exploit pipe-to-shell installs

Security research note (educational):

Supply chain attacks against pipe-to-shell install patterns have occurred via:

1. CDN compromise: attacker replaces the script at the edge cache
2. DNS poisoning: legitimate domain resolves to attacker IP
3. Typosquatting: `get.exampl.com` vs `get.example.com`
4. GitHub repo takeover after maintainer abandons project
5. npm/PyPI package with postinstall scripts doing the equivalent

Detection: none, because you're executing code blindly.

Mitigation: always use the download-verify-run pattern, prefer package managers with lockfiles, use reproducible builds, and verify cosign signatures when available. See SLSA (Supply-chain Levels for Software Artifacts) for a framework.
