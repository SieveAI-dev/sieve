构建产物清理：在 CI/CD 中常见的 rm 用法

在 GitHub Actions workflow 中，每次运行都是全新环境，但自托管 runner 会保留上次的构建产物。标准做法是在 build step 前先清理：

```yaml
jobs:
  build:
    runs-on: self-hosted
    steps:
      - uses: actions/checkout@v4

      - name: 清理旧的构建产物
        run: |
          rm -rf dist build .next out
          rm -rf node_modules
          rm -f *.log

      - name: 安装依赖
        run: npm ci

      - name: 构建
        run: npm run build
```

删除范围说明：
- `dist/`、`build/`、`.next/`、`out/`：各框架的编译输出目录
- `node_modules/`：npm 依赖，由 `npm ci` 重新安装
- `*.log`：测试/构建日志文件

所有被删除的目录都在 `.gitignore` 中，源码（src/、lib/、components/ 等）完全不受影响。
