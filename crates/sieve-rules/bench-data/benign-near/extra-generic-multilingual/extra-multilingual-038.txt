Comment optimiser les performances d'une API Node.js sous forte charge ?

Mon API Express.js traite environ 500 req/s et j'observe des latences élevées (p99 > 2s) lors des pics.

```javascript
app.get('/api/data', async (req, res) => {
  const result = await db.query('SELECT * FROM events WHERE user_id = $1', [req.user.id]);
  res.json(result.rows);
});
```

Les pistes que j'envisage :
1. Connection pooling (j'utilise `pg`, est-ce que `pg-pool` est nécessaire ?)
