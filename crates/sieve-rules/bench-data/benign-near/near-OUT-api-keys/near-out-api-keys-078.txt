GCP API key restrictions — how to lock down an AIza... key:

In the GCP Console → APIs & Services → Credentials → click your key:

**Application restrictions** (choose one):
- None (unrestricted — avoid for production)
- HTTP referrers: `*.myapp.com/*`
- IP addresses: `203.0.113.0/24` (server-side use)
- Android apps: package name + SHA-1 cert fingerprint
- iOS apps: bundle ID

**API restrictions** (always set this):
- Restrict key → select only the APIs this key needs
- E.g., Maps JavaScript API, Places API

An unrestricted AIza... key is a billing liability if exposed.
