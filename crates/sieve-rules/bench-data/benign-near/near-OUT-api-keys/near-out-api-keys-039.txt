AWS IAM best practices for access keys:

1. Prefer IAM roles over long-term AKIA access keys wherever possible
2. If you must use access keys, create them for service accounts not for human users
3. Enable MFA for all human IAM users
4. Rotate AKIA keys every 90 days: aws iam create-access-key && update services && aws iam delete-access-key
5. Never store AKIA keys in code, git history, or environment variables on shared machines
6. Use AWS Secrets Manager or Parameter Store for secrets in Lambda/ECS
7. Enable CloudTrail to audit key usage
8. Set up AWS Config rule `access-keys-rotated` to alert on old keys
