AWS credential provider chain in SDK (Python boto3):

boto3 searches for credentials in this order:
1. Explicit parameters in client constructor
2. Environment variables (AWS_ACCESS_KEY_ID=AKIA...)
3. AWS config file (~/.aws/config)
4. AWS credentials file (~/.aws/credentials)
5. AWS SSO session (if configured)
6. Container credential provider (ECS)
7. EC2 instance metadata (IMDS)

For Lambda and ECS, use execution roles (no AKIA keys needed). For local dev, use `aws configure` or AWS SSO.
