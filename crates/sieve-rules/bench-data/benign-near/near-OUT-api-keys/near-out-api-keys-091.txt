StackOverflow-style Q&A: Multiple API keys in one project

Q: I'm building an app that uses Anthropic, OpenAI, and AWS S3. How do I manage all these keys safely?

A: Use environment variables for all of them. Create a .env file (never committed):

```
ANTHROPIC_API_KEY=sk-ant-api03-REPLACE_ME
OPENAI_API_KEY=sk-proj-REPLACE_ME
AWS_ACCESS_KEY_ID=AKIA_REPLACE_ME
AWS_SECRET_ACCESS_KEY=REPLACE_ME
GITHUB_TOKEN=ghp_REPLACE_ME
GOOGLE_API_KEY=AIzaSy-REPLACE_ME
```

Load with python-dotenv or direnv. In production, use a proper secrets manager. Never commit the actual values.
