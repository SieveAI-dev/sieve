GCP API Key vs Service Account — when to use each:

**API Key (AIza...):**
- Simple authentication for public APIs (Maps, YouTube, Translate)
- No identity — anyone with the key can call the API
- Restrict by API type + HTTP referrer or IP
- Best for: client-side JS, mobile apps with restricted scope

**Service Account:**
- Full GCP identity with IAM roles
- Authenticates with JSON key file or workload identity
- Best for: server-side code, GCP resource management

Never use a service account JSON key where an API key (AIza...) would suffice — principle of least privilege.
