## JWT Authentication Flow (Node.js / Express)

1. User logs in with credentials.
2. Server validates, calls `jwt.sign({ userId: user.id }, process.env.JWT_SECRET, { expiresIn: '1h' })`.
3. Client stores the token (localStorage or httpOnly cookie).
4. On subsequent requests, client sends `Authorization: Bearer <token>`.
5. Middleware calls `jwt.verify(token, process.env.JWT_SECRET)`.

Never hardcode secrets. Use environment variables.

Sample token shape for documentation:
  eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJzdWIiOiJFWEFNUExFX1JFUExBQ0VfTUUiLCJpYXQiOjE1MTYyMzkwMjJ9.EXAMPLE_REPLACE_ME

Replace `EXAMPLE_REPLACE_ME` with your actual signed token in production testing.
