Discord Bot Token: Safe Placeholder Formats for Documentation

When writing tutorials, use these safe formats:

For Discord bot tokens:
  EXAMPLE.REPLACE.MEDONOTUSEINTHISCODE
  YOUR_BOT_TOKEN_HERE
  <YOUR_DISCORD_BOT_TOKEN>
  process.env.DISCORD_BOT_TOKEN   ← show the env var reference

For .env examples:
  DISCORD_BOT_TOKEN=EXAMPLE.REPLACE.MEDONOTUSEINTHISCODE

Real token format: [24-28 chars].[6 chars].[27-38 chars] of base64url.
A placeholder like EXAMPLE.REPLACE.ME won't match real segment lengths.

Never use a real token in documentation—even inactive bots retain valid tokens
until explicitly regenerated. Rotate any token that has been exposed.
