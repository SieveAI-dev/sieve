## Slack App Distribution: Public vs Internal

Internal Apps:
  - For your own workspace only
  - No Slack App Directory review
  - Direct installation, immediate token grant
  - Token: xoxb-... scoped to your workspace

Public Distribution Apps:
  - Can be installed by any Slack workspace
  - Requires Slack App Directory review
  - Must implement OAuth 2.0 flow
  - Each installation gets unique xoxb-... token per workspace

OAuth installation flow:
  1. User clicks "Add to Slack" button
  2. Redirect to: https://slack.com/oauth/v2/authorize?client_id=...&scope=chat:write
  3. User approves
  4. Slack redirects to your callback URL with `code`
  5. Exchange code for token: POST https://slack.com/api/oauth.v2.access

Store each workspace's token securely (per-team), keyed by team_id.
