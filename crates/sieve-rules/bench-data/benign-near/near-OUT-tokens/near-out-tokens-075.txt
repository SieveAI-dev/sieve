# Slack Automation: Workflow Builder vs Bolt

Workflow Builder (no-code):
  - Create workflows triggered by Slack events
  - No programming required
  - Limited to built-in steps and third-party integrations
  - Workflow tokens managed by Slack, not exposed to developers

Bolt Framework (code):
  - Full programmatic control over events, commands, modals
  - Requires bot token (xoxb-...) stored securely
  - Supports Socket Mode for local development

Hybrid: Register a Bolt step callable from Workflow Builder.

```python
from slack_bolt import App
app = App(token=os.environ.get("SLACK_BOT_TOKEN"))

@app.step("my_step")
def handle_step(ack, complete, inputs):
    ack()
    complete(outputs={"result": inputs["val"].upper()})
```

Token via SLACK_BOT_TOKEN (xoxb-EXAMPLE-... for docs).
