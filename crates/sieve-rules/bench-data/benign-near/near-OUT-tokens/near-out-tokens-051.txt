# Slack Bot Token Types

Slack issues several token types for different use cases:

xoxb-* (Bot tokens): Used by Slack apps with bot user capabilities.
  Granted when app is installed to a workspace. Required for most bot operations.

xoxp-* (User tokens): Act as a specific user in a workspace.
  Require user OAuth. Rare in modern Slack apps—prefer bot tokens.

xoxa-* (App-level tokens): Used for Socket Mode connections.
  Not workspace-specific. Starts with xapp- in newer apps.

xoxs-* (Workspace tokens): Legacy. Deprecated.

For new apps, use Bot tokens (xoxb-EXAMPLE-...).
Store tokens in environment variables, never hardcode.
Rotate tokens if compromised via Slack App Dashboard → Features → OAuth & Permissions.
