# Discord Bot Token Security

Token format breakdown (educational, not a real token):
  [UserID_base64].[Timestamp_base64].[HMAC_base64]

Part 1 (24-28 chars): Encodes the bot's User ID (snowflake)
Part 2 (6 chars): Millisecond timestamp since Discord epoch
Part 3 (27-38 chars): HMAC-SHA256 signature

Do NOT decode real tokens to extract user IDs—use the API instead:
  GET https://discord.com/api/v10/users/@me
  Authorization: Bot YOUR_TOKEN

Security checklist:
  [ ] Token stored in environment variable
  [ ] .env file in .gitignore
  [ ] Token not logged or returned in error messages
  [ ] Minimum required intents and permissions
  [ ] Token regenerated after any suspected exposure

Discord will automatically revoke tokens detected in public git repositories.
