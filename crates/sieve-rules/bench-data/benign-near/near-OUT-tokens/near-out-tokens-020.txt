## JWT in Microservices: Key Distribution

Option A: Shared secret (HS256)
- All services share the same JWT_SECRET
- Simple, but compromise of one service exposes all

Option B: Asymmetric keys (RS256 / ES256)
- Auth service holds private key; others get the public key
- Compromise of downstream service does NOT expose private key
- Public key distributed via JWKS endpoint

Option C: API Gateway validates JWT
- Edge validates; downstream services trust forwarded headers
- Reduces crypto overhead in backend services
- Requires secure internal network (mTLS or VPN)

Recommendation: Option B for production.
JWKS endpoint returns public keys only:
  {"kty":"RSA","kid":"v1","use":"sig","n":"EXAMPLE_N_VALUE","e":"AQAB"}

Use kid (key ID) in JWT header to support key rotation without downtime.
