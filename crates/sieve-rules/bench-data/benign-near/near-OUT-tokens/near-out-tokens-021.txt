# Troubleshooting: JWT exp claim timezone issue

Problem: Tokens expire immediately after creation.

Root cause: Mixing seconds and milliseconds for the exp claim.

Node.js jsonwebtoken uses seconds:
  jwt.sign({ sub: '42' }, secret, { expiresIn: '1h' })
  // exp = Math.floor(Date.now() / 1000) + 3600  ✓

Python PyJWT uses seconds:
  jwt.encode({"sub": "42", "exp": datetime.utcnow() + timedelta(hours=1)}, key)  ✓

Java JJWT:
  Jwts.builder().setExpiration(Date.from(Instant.now().plusSeconds(3600)))  ✓

Debug: Paste the token at jwt.io to inspect the exp claim value.
