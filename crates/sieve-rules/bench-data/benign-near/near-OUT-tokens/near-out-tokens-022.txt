# JWT Logout: Why it's complicated

JWTs are stateless by design—once issued, they're valid until expiry.
To "log out" a user, you have several options:

1. Short expiry (15 min): User is effectively logged out quickly without extra infra.
2. Refresh token revocation: Keep refresh tokens in DB; revoking one prevents new access tokens.
3. Token blocklist: Store revoked JTI (jti claim) in Redis until token expires.
4. Version field: Store user's token_version in DB; increment on logout. Include version in JWT payload and validate on each request.

Option 4 example payload: {"sub":"usr_123","tv":7,"exp":1700000000}
The server checks: if user.token_version != jwt.tv, reject.

Trade-off: all options re-introduce statefulness. Choose based on your revocation latency needs.
