# Stripe API Key Rotation Procedure

When to rotate live keys:
- After any suspected compromise
- When an employee with key access leaves
- As part of regular security review (quarterly recommended)

Rotation steps:
1. Create new key in Stripe Dashboard → Developers → API Keys → Create key
2. Update secrets in your secrets manager (AWS Secrets Manager, Vault, etc.)
3. Deploy updated applications without downtime (old key still valid)
4. Monitor error rates to confirm new key works
5. Revoke old key in Stripe Dashboard

During transition, both old and new keys are valid simultaneously.
After revocation, requests using the old key return 401 Unauthorized.

Keep an audit log: key_id, created_at, rotated_at, rotated_by, reason.
