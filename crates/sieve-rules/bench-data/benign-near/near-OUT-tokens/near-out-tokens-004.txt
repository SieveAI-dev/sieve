Common JWT validation errors and their causes:

TokenExpiredError: The `exp` claim is in the past. Solution: issue a new token.
JsonWebTokenError: Signature verification failed. Check that client and server use the same secret/key.
NotBeforeError: The `nbf` claim is in the future. Token not yet valid.

Test with a known-invalid token to verify your error handler works:
  const badToken = "eyJhbGciOiJub25lIn0.eyJzdWIiOiJ0ZXN0In0.";
  jwt.verify(badToken, secret);  // Should throw JsonWebTokenError

Never catch these errors silently. Log them with a request ID for auditing.
