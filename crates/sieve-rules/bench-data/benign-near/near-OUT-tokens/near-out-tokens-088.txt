# Discord Webhook vs Bot Token

Incoming Webhooks:
  URL: https://discord.com/api/webhooks/{id}/{token}
  Use: One-way, post to a specific channel
  No bot account required
  Limited: Can't read messages, react, or perform moderation

Bot Token (multi-part token):
  Full API access via Discord gateway
  Can read, write, react, manage channels/roles
  Requires bot account registration

When to use webhooks:
  - CI/CD notifications (GitHub Actions → Discord)
  - Monitoring alerts
  - Simple one-way integrations

When to use bot token:
  - Interactive commands
  - Moderation features
  - Reading and responding to messages
  - Server management

Webhook token is embedded in the URL—treat the entire URL as a secret.
Bot token is a separate credential managed in the Developer Portal.
