# Slack app in Docker: secure token injection

```dockerfile
FROM python:3.12-slim
WORKDIR /app
COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt
COPY . .
# Do NOT copy .env into the image
CMD ["python", "app.py"]
```

```yaml
# docker-compose.yml
services:
  slack-bot:
    build: .
    environment:
      - SLACK_BOT_TOKEN  # Value comes from host environment, not the compose file
      - SLACK_SIGNING_SECRET
```

Run: SLACK_BOT_TOKEN=xoxb-... docker compose up

For production: use Docker secrets or inject via your orchestrator (K8s Secrets, ECS task def).
Never embed xoxb-... tokens in Docker images or compose files committed to git.
