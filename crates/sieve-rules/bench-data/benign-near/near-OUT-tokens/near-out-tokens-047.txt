Stripe API Versioning

Stripe uses date-based versioning: 2024-06-20, 2023-10-16, etc.

Your account has a default API version set in the Dashboard.
You can override per-request:

```python
stripe.api_key = os.environ["STRIPE_SECRET_KEY"]  # sk_test_* in dev

# Override version for a single request
response = stripe.checkout.Session.create(
    ...,
    stripe_version="2024-06-20"
)
```

When upgrading:
1. Read the changelog for breaking changes
2. Test with sk_test_* key against the new version
3. Update your Stripe-Version header in the Dashboard
4. Monitor for errors in production

Breaking changes are announced 90 days in advance.
Older versions are supported for years (check Stripe's versioning policy).
