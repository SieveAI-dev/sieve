Managing authorized_keys on a remote server:

File location: ~/.ssh/authorized_keys
Permissions: chmod 600 ~/.ssh/authorized_keys (must be strict)

Add a public key (append):
  echo "ssh-ed25519 AAAAC3Nz... user@host" >> ~/.ssh/authorized_keys

Or use ssh-copy-id (copies your local public key to remote):
  ssh-copy-id -i ~/.ssh/id_ed25519.pub user@remotehost

Restrict a key with options (prefix in authorized_keys):
  from="10.0.0.0/8",command="/usr/local/bin/backup.sh",no-pty ssh-ed25519 AAAA...

Revoke access: remove the specific line from authorized_keys. There is no CRL equivalent for SSH keys — revocation is manual file editing.
