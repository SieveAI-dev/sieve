TLS certificate pinning: embedding a certificate's public key fingerprint to prevent MITM attacks. Pins can be the leaf cert or a CA cert (more resilient to rotation).

Calculate SPKI pin for a certificate (safe operation using only public data):
  openssl x509 -in cert.pem -pubkey -noout | \
    openssl pkey -pubin -outform DER | \
    openssl dgst -sha256 -binary | base64

This outputs the SHA-256 hash of the SubjectPublicKeyInfo — what browsers use for HPKP (now deprecated) and what mobile apps use for certificate pinning.

In nginx (modern approach using ssl_trusted_certificate):
  ssl_trusted_certificate /etc/nginx/ssl/chain.pem;
  
Pinning with certificate fingerprint does not expose any private key material. The pin is derived entirely from the public certificate.
