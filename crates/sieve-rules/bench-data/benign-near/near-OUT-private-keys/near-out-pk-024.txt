Managing ~/.ssh/known_hosts:

List all known hosts:
  ssh-keygen -l -f ~/.ssh/known_hosts

Remove a stale host entry (after server key rotation):
  ssh-keygen -R hostname.example.com
  ssh-keygen -R 10.0.1.50

Hash host entries for privacy (hides hostnames):
  ssh-keygen -H -f ~/.ssh/known_hosts

Scan and add host key before connecting (verify out-of-band):
  ssh-keyscan -H hostname.example.com >> ~/.ssh/known_hosts

"REMOTE HOST IDENTIFICATION HAS CHANGED" error means the server's host key changed — verify intentional (e.g. server rebuild) before accepting new key. Never blindly delete the entry if you didn't expect a change.
