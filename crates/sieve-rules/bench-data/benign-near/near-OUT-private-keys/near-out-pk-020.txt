Private key storage best practices (do NOT commit keys to git):

1. Use hardware security modules (HSM) or TPM for production keys
2. Store keys in secrets managers: AWS Secrets Manager, HashiCorp Vault, GCP Secret Manager
3. If using files: strict permissions (chmod 600 on Unix), owned by service user
4. Consider PKCS#11 interface to access keys in HSM without exporting them
5. For CI/CD: inject as environment variables or mounted secrets, never bake into container images
6. Key rotation: automate with cert-manager (Kubernetes) or ACM (AWS)
7. Audit access: log every time a private key is used for signing operations
8. Never log, print, or transmit key material in plaintext
