SSH and TLS key file permission requirements:

SSH client enforces strict permissions:
  ~/.ssh/id_ed25519      must be 600 (rw-------)
  ~/.ssh/config          must be 600 or 644
  ~/.ssh/                must be 700 (rwx------)
  ~/.ssh/authorized_keys must be 600

"Permissions 0644 for 'id_ed25519' are too open" → chmod 600 ~/.ssh/id_ed25519

sshd enforces similar rules on server:
  /etc/ssh/ssh_host_*_key  must be 600, owned by root

For TLS private keys in nginx/apache:
  /etc/nginx/ssl/privkey.pem should be 600, owned by root
  nginx reads it as root before dropping privileges

Fix permissions recursively:
  chmod 700 ~/.ssh
  find ~/.ssh -type f -exec chmod 600 {} \;
  find ~/.ssh -name "*.pub" -exec chmod 644 {} \;
