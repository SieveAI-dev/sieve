Anatomy of a PEM file: The header "-----BEGIN <LABEL>-----" and footer "-----END <LABEL>-----" are literal ASCII dashes (five on each side) with the label in between. The body is base64-encoded DER data with lines wrapped at 64 characters. Multiple PEM objects can appear in one file (a "PEM bundle" or "chain file"). For TLS, a chain file contains the leaf certificate followed by intermediate CAs. Order matters: leaf first, root last (or omitted since clients have it). Concatenate with: cat cert.pem intermediate.pem > chain.pem
