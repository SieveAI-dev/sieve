Modern OpenSSH private key format (since OpenSSH 6.5) uses a custom binary format, not standard PKCS#8. The file header is "BEGIN OPENSSH PRIVATE KEY" followed by base64 of a proprietary structure containing:

- Magic: "openssh-key-v1\0"
- ciphername: "none" (unencrypted) or "aes256-ctr" (encrypted)
- kdfname: "none" or "bcrypt"
- kdfoptions: bcrypt salt and rounds
- public key section
- private key section (encrypted if passphrase set)

This format supports passphrases with bcrypt KDF, which is more memory-hard than the older PKCS#1 format's simple MD5 hashing. Convert to PKCS#8 PEM (if needed for legacy tools): ssh-keygen -p -m PEM -f ~/.ssh/id_rsa
