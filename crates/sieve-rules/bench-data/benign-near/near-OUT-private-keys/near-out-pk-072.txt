SSH key revocation — how to respond to a potentially compromised key:

Immediate response (remove from all servers):
1. Remove from each server's authorized_keys:
   ssh user@server "sed -i '/COMPROMISED_KEY_COMMENT/d' ~/.ssh/authorized_keys"

2. If using SSH CA: add to Key Revocation List:
   ssh-keygen -k -f /etc/ssh/revoked_keys -z 1 revoked_pubkey.pub
   # In sshd_config: RevokedKeys /etc/ssh/revoked_keys

3. Remove from GitHub/GitLab/Bitbucket SSH settings immediately

4. Remove from CI/CD systems (GitHub Actions secrets, GitLab CI variables)

5. Generate new key pair:
   ssh-keygen -t ed25519 -C "replacement-$(date +%Y%m%d)"

6. Update authorized_keys on all servers with new public key

7. Check git commits that may have used the old key for GPG signing — if GPG, revoke GPG key separately

Document the incident. Audit what the key had access to.
