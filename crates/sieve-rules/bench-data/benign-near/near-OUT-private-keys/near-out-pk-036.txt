SSH Certificate Authorities (CA) allow fleet-wide key management without updating authorized_keys on every host. A CA key signs user or host keys; servers trust the CA instead of individual keys.

Create CA key pair:
  ssh-keygen -t ed25519 -f ssh_ca -C "SSH CA for example.com"
  # ssh_ca (keep secret) and ssh_ca.pub (distribute to servers)

Sign a user's public key:
  ssh-keygen -s ssh_ca -I "username" -n "ubuntu,ec2-user" -V +52w user_pubkey.pub
  # Creates user_pubkey-cert.pub (the certificate)

Configure server to trust CA (in /etc/ssh/sshd_config):
  TrustedUserCAKeys /etc/ssh/ssh_ca.pub

User presents cert + private key. Cert validity period is automatically enforced. Revoke by expiring the cert or adding to KRL (Key Revocation List).
