Verifying SSH host key fingerprints before first connection:

When connecting to a new server:
  The authenticity of host 'example.com (203.0.113.42)' can't be established.
  ED25519 key fingerprint is SHA256:abc123xyz...
  Are you sure you want to continue connecting (yes/no/[fingerprint])?

Verify by obtaining fingerprint through a secure out-of-band channel:
- Cloud provider console (AWS EC2 system log, GCP serial console)
- Your infrastructure-as-code (Terraform/Ansible creates the key, records fingerprint)
- Direct access via out-of-band management interface

Get server fingerprint before connecting (if you have other access):
  sudo ssh-keygen -l -f /etc/ssh/ssh_host_ed25519_key.pub
  sudo ssh-keygen -lv -f /etc/ssh/ssh_host_ed25519_key.pub  # randomart version

Never blindly accept "yes" for a new host. The "yes" caches the key in known_hosts; subsequent connections detect key changes automatically.
