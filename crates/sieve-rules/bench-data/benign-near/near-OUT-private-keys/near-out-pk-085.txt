Ansible Vault for secrets management — alternative to storing PEM keys in plaintext:

Encrypt a file containing sensitive content:
  ansible-vault encrypt secrets.yml
  # Uses AES-256-CBC with a vault password (not PEM key)

Encrypt a single value (for embedding in playbooks):
  ansible-vault encrypt_string 'secret-value' --name 'db_password'

Decrypt for inspection:
  ansible-vault view secrets.yml
  ansible-vault decrypt secrets.yml --output decrypted.yml

For SSH private keys in Ansible: reference key file path in playbooks:
  ansible_ssh_private_key_file: ~/.ssh/id_ed25519
  # Or in inventory: host ansible_ssh_private_key_file=/path/to/key

Never put actual private key content inline in Ansible playbooks. Reference the file path or use a secrets manager like HashiCorp Vault with the hashi_vault lookup plugin.
