ACME protocol (RFC 8555) — how Let's Encrypt automates certificate issuance:

1. Client generates a key pair (private/public) for the ACME account
2. Client proves domain control via HTTP-01 or DNS-01 challenge:
   - HTTP-01: places a token at http://domain/.well-known/acme-challenge/<token>
   - DNS-01: creates a TXT record at _acme-challenge.domain with a token value
3. ACME server verifies the challenge
4. Client submits a CSR (certificate signing request) containing only the public key
5. Server returns a signed certificate (BEGIN CERTIFICATE block)

The private key never leaves your server. The CSR and certificate are public. ACME client implementations: certbot, acme.sh, caddy (built-in), cert-manager (k8s). RFC 8555 specifies the entire protocol; private keys are never transmitted.
