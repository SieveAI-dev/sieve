# SSH Security Hardening Checklist

## Key Management
- [ ] Use Ed25519 keys (not RSA-1024 or DSA)
- [ ] Set passphrase on private keys
- [ ] Set correct permissions: `chmod 600 ~/.ssh/id_*` (not .pub files)
- [ ] One key per device, not shared across machines
- [ ] Separate deploy keys for CI/CD (scoped, no passphrase for automation)

## Server Configuration (/etc/ssh/sshd_config)
- [ ] `PasswordAuthentication no`
- [ ] `PermitRootLogin no`
- [ ] `MaxAuthTries 3`
- [ ] `AllowUsers` or `AllowGroups` specified
- [ ] `Protocol 2` (SSH protocol 1 is broken)

## Monitoring
- [ ] Log all SSH logins: check /var/log/auth.log or journalctl -u ssh
- [ ] Set up fail2ban or similar to block brute force
- [ ] Alert on new authorized_keys additions

## Rotate keys when
- [ ] Employee offboarding
- [ ] Device lost or compromised
- [ ] Key exposed in logs/commits
