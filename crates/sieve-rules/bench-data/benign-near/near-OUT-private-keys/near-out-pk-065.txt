GPG vs SSH keys — different purposes, similar PEM confusion:

SSH keys: authentication only. Public key format: "ssh-ed25519 AAAA..." or "-----BEGIN PUBLIC KEY-----". Private key format stored in OpenSSH native format.

GPG keys: signing, encryption, authentication. Stored in ~/.gnupg/ as binary keyring files, NOT in PEM format. Export to ASCII-armored format:
  gpg --armor --export user@example.com > pubkey.asc
  # Output: -----BEGIN PGP PUBLIC KEY BLOCK----- (public, safe to share)

  gpg --armor --export-secret-keys user@example.com > secret.asc
  # Output uses the OpenPGP armor header for secret keys (keep secret)

Note: OpenPGP secret key armor uses "PGP" in its label, distinct from PKCS#8 ("PRIVATE KEY") or OpenSSH format. Each format is tool-specific and not interchangeable. The OpenPGP format is only used by gpg/pgp tooling, not by OpenSSL or ssh-keygen.
