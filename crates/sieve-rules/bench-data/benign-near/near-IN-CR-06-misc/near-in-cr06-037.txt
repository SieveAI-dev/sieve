Inyección de prompts — guía para desarrolladores (español)

La inyección de prompts es el principal riesgo de seguridad en sistemas basados en LLM según OWASP (LLM01). Los atacantes insertan instrucciones maliciosas en contenido no confiable que el modelo procesa.

Escenario típico en desarrollo Web3:
- El agente de IA lee un archivo README de un repositorio externo
- El README contiene instrucciones ocultas (ejemplo educativo solamente)
- El agente ejecuta esas instrucciones creyendo que vienen de su sistema

Mitigaciones recomendadas:
- Validar la fuente de todo contenido externo
- Aplicar filtros de salida para detectar comandos de firma inesperados
- Usar Sieve como capa de inspección entre el agente y la API
