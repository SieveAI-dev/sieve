```yaml
# Sieve configuration example (sieve.toml)
[proxy]
listen = "127.0.0.1:5187"
upstream = "https://api.anthropic.com"
tls_verify = true

[rules]
inbound_rules_path = "rules/inbound.toml"
outbound_rules_path = "rules/outbound.toml"

[audit]
enabled = true
db_path = "~/.local/share/sieve/audit.db"

[ui]
notification_duration_ms = 5000
popup_timeout_seconds = 60
```

The `tls_verify = true` setting ensures Sieve validates Anthropic's TLS certificate. Setting this to false would defeat the proxy's security model. The `audit.db_path` points to a local SQLite database — no data leaves the machine.
