Cognitive friction as a security mechanism

Behavioral security research shows that adding deliberate friction to high-stakes actions reduces errors and prevents manipulation. Examples:
- Bank transfers requiring a second confirmation
- Deleting a GitHub repo requiring you to type the repo name
- Two-factor authentication adding a second step

Sieve applies this principle to AI-assisted development: before an irreversible blockchain action (sign, transfer, deploy), it inserts a confirmation dialog. The goal is not to make the action impossible but to ensure the user is conscious of what's about to happen.

A 5-second pause significantly reduces accidental or manipulated actions. The friction is calibrated to be minimal for legitimate use but effective against automated attack chains.
