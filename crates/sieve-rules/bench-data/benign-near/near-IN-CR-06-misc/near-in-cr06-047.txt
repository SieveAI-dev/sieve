Sieve IPC protocol overview

Sieve uses a file-based IPC protocol for hook ↔ GUI communication:

1. Hook writes a pending decision to ~/.local/share/sieve/pending/{uuid}.json
2. GUI reads the pending file, shows the user a confirmation popup
3. User approves or denies
4. GUI writes the decision to ~/.local/share/sieve/decisions/{uuid}.json
5. Hook reads the decision file and returns the result to Claude Code

File locking (fd-lock) prevents race conditions. The hook has a 30-120s timeout depending on rule severity; on timeout it applies the `default_on_timeout` value (always "block" for Critical rules).

This design avoids a Unix socket listener in the hook process, keeping startup latency under 50ms.
