Threat model: what Sieve protects against and what it doesn't

**In scope:**
- Prompt injection attacks delivered via LLM responses
- Accidental credential exfiltration in prompts
- AI agent misuse of crypto signing tools
- Dynamic skill/plugin supply chain attacks
- Shell persistence attacks initiated by AI agent

**Out of scope (Phase 1):**
- Attacks on the Sieve binary itself (supply chain addressed by sigstore)
- Network MITM attacks (no local CA, by design — see PRD §9 #12)
- Attacks on the Claude Code binary
- Social engineering of the human user directly (not computer-mediated)
- Attacks against the Anthropic API backend

Understanding the scope helps avoid both over-relying on Sieve for things it doesn't protect against and under-valuing it for things it does.
