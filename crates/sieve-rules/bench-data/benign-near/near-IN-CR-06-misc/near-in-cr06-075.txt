Sieve architecture: five-crate design

```
sieve-core     — SSE parser, UnifiedMessage, pipeline, forwarder
sieve-rules    — rule definitions, vectorscan engine, Ed25519 verification
sieve-cli      — entry point, config, audit log (SQLite), macOS/Linux daemon
sieve-ipc      — IPC protocol (JSON-RPC over Unix socket, file-based decisions)
sieve-hook     — minimal PreToolUse hook binary (<50ms startup, minimal deps)
```

Dependency constraints (enforced by Cargo):
- sieve-hook must NOT depend on sieve-core, sieve-rules, or tokio
- sieve-rules must NOT have network I/O
- sieve-core must NOT have CLI or config loading

This separation ensures the hook stays fast, the rules stay testable in isolation, and the core stays focused on pipeline logic.
