Prompt injection in multi-agent systems

When multiple AI agents communicate, the attack surface multiplies. Agent A might fetch a webpage, extract text, and pass it to Agent B. If that text contains injection payloads, Agent B receives what looks like a trusted message from Agent A.

Mitigations:
- Label all inter-agent messages with source provenance
- Apply filtering at every agent boundary, not just entry points
- Use capability tokens: Agent B cannot sign or transfer unless it received an explicit user-approved capability delegation

This is exactly what Sieve's IPC protocol implements: each inter-agent decision goes through the pending/decisions file, not the network.
