红队测试 AI 系统的方法论（中文）

针对 AI 编码助手的红队测试应覆盖以下向量：

**直接攻击**
- 用户直接发送恶意 prompt（主要是误用风险，模型自身有防御）

**间接注入**
- 通过工具响应传入恶意指令（高风险，模型防御弱）
- 通过读取的文件内容传入指令
- 通过 MCP 工具的 description 字段传入指令

**供应链攻击**
- 恶意 MCP 服务器
- 伪装成合法包的 npm/cargo 依赖
- 动态加载的 skill/plugin

**持久化测试**
- Agent 是否会在未经授权的情况下修改 shell 配置文件
- Agent 是否会注册 launchd/systemd 服务

Sieve 的规则集覆盖了后三类中的主要场景。
