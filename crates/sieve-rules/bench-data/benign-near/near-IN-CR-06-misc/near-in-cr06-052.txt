SSE streaming and security filtering

Server-Sent Events (SSE) deliver data as a stream of newline-delimited events. For LLM APIs, each event is a JSON delta containing a fragment of the model's response.

Security filtering on SSE is harder than on complete responses because:
- An attack pattern might span multiple chunks
- Filtering must be stateful across chunk boundaries
- Blocking mid-stream requires careful handling to avoid corrupting the client

Sieve's SSE parser maintains state across chunk boundaries using a finite state machine. When a match completes, Sieve holds the stream until the event is fully parsed, sending keep-alive comments to prevent client timeout.
