IN-GEN-04 benign variant: analytics with no sensitive data

Marketing and documentation sites commonly use image-based analytics:

```markdown
![Page view](https://analytics.mysite.com/pixel?page=docs&section=getting-started)
```

This URL has query parameters, but they contain no user data — just page metadata. Sieve's IN-GEN-04 rule would flag this because it has query parameters. This is an expected false positive category. The allowlist mechanism can be used to exempt known analytics domains:

```toml
allowlist_regexes = ['https?://analytics\\.mysite\\.com/pixel']
```

The rule errs on the side of caution because the pattern is identical to an exfiltration URL — only the semantics of the parameter values differ, which pattern matching cannot assess.
