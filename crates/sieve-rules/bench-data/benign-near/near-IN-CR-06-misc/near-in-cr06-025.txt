Sieve detection item IDs — what they mean

Sieve uses a structured ID scheme for detection items:

OUT-NN: Outbound detections (data leaving toward the API)
  OUT-01: Anthropic API key pattern
  OUT-02: AWS access key
  OUT-05: BIP39 mnemonic phrase
  OUT-12: Ethereum private key (hex)

IN-CR-NN: Inbound Critical rules (high-confidence attacks, default block)
  IN-CR-01: Address substitution (Levenshtein-based)
  IN-CR-02: Dangerous shell commands
  IN-CR-05: Crypto signing tool calls
  IN-CR-06: OpenClaw dynamic skill installation

IN-GEN-NN: Inbound General rules (lower confidence, may warn vs. block)
  IN-GEN-01: Markdown javascript: URI
  IN-GEN-04: Markdown image with data-bearing query params
  IN-GEN-06: External prompt injection phrases
