Sieve's approach to outbound PII redaction

Automatic redaction (no user prompt):
- Pattern matches → replace with [REDACTED_TYPE] in the request body
- Status bar notification: "Sieve: redacted API key from outbound request" (5 seconds, dismissible)
- Audit log entry with: timestamp, rule ID, truncated context (50 chars before/after)

No popup dialog for routine redaction. The design principle is that users should not be interrupted dozens of times per day for automated fixes. Popups are reserved for cases where:
- A Critical inbound event requires explicit user decision
- Sieve cannot automatically determine the safe action

This matches PRD §9 #13: "outbound redaction must not interrupt workflow."
