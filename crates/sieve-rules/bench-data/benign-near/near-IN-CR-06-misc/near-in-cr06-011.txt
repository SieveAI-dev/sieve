Claude Code Permission System Overview

Claude Code uses a layered permission model:
- User-level permissions: ~/.claude/settings.json
- Project-level permissions: .claude/settings.json (committed to repo)
- Tool-level: each MCP tool declares required permissions

The `allowedTools` list controls which tools Claude can call without prompting. Tools not in the list require explicit user approval. Sensitive tools (bash, file write, network) should require explicit approval by default.

Sieve's PreToolUse hook adds a third layer: even if Claude Code would allow a tool, Sieve can block it based on the content of the inputs, not just the tool name.
