AI coding tools comparison from a security perspective

| Tool | Hook support | Custom base URL | Open source | Offline capable |
|------|-------------|----------------|-------------|-----------------|
| Claude Code | Yes (PreToolUse/PostToolUse) | Yes (ANTHROPIC_BASE_URL) | Partially | No |
| OpenClaw | Yes (skill system) | Yes | Plugin-based | Partial |
| Hermes | Limited | Yes (OPENAI_BASE_URL) | No | No |
| Cursor | No hooks | Via proxy config | No | No |
| GitHub Copilot | No hooks | No | No | No |

For Sieve integration, the critical requirement is "custom base URL" support — without it, traffic cannot be routed through the proxy. Hook support enables the dual-layer defense (SSE-level + hook-level); without hooks, only SSE filtering is available.
