Understanding Sieve's allowlist mechanism

Sieve rules can have an `allowlist_regexes` field that exempts matches from triggering the rule. Example from IN-CR-03-SSH-DIR:

```
allowlist_regexes = ['~/\\.ssh/(?:known_hosts|authorized_keys|config|environment)\\b']
```

This means: even if the rule's main pattern matches `~/.ssh/...`, if the full string also matches the allowlist regex, the match is suppressed.

Allowlists are the right tool for handling legitimate edge cases without deleting rules. If Sieve causes a false positive in your workflow, check whether an allowlist entry would fix it before asking for the rule to be weakened or removed.
