Sieve rule severity levels explained

Critical: block immediately, no user override, fail-closed
  - Used for: wallet signing, dangerous shell commands, address substitution, dynamic skill install

High: warn user, allow override, 30s timeout then block
  - Used for: sensitive file access (~/.ssh, .env), markdown image exfil patterns

Medium: log and notify, non-blocking
  - Used for: borderline patterns where false positive rate is higher

The severity assignment is documented in the inbound.toml rule file. Changing a rule from High to Critical is a breaking change (requires ADR and CHANGELOG entry) because it affects user workflow.
