Sieve and the "agentic turn" in software development

AI coding tools are rapidly evolving from "autocomplete" to "autonomous agents." This shift creates new security properties to reason about:

Autocomplete era:
- Model suggests code, human types it
- Human is always in the loop before any action
- Security responsibility: entirely human

Agent era:
- Model reads code, runs tests, edits files, calls APIs, commits changes
- Human may review asynchronously or not at all (YOLO mode)
- Security responsibility: shared between human and runtime controls

Sieve targets the agent era specifically. The need for a runtime security layer did not exist when models could only suggest text.
