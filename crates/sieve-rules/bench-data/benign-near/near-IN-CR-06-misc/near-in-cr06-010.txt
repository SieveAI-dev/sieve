PreToolUse hooks in Claude Code

Claude Code supports hooks that run before and after tool calls. A PreToolUse hook receives the tool name and input as JSON on stdin, and can output a decision (approve/block/modify) on stdout.

This is the interception point Sieve uses. The hook binary is registered in ~/.claude/settings.json:

{
  "hooks": {
    "PreToolUse": [{"command": "sieve-hook"}]
  }
}

The hook reads the IPC pending file, waits for the GUI decision, and returns the result — all within the tool call timeout. This design keeps Sieve out of the network path for hook-type decisions.
