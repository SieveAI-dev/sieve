Legitimate use of Markdown image syntax

Images in Markdown documentation are normal and useful:
![Architecture diagram](https://docs.example.com/images/arch.png)
![Build status](https://github.com/org/repo/actions/workflows/ci.yml/badge.svg)

These are fine — the URL has no query parameters carrying data. The concerning pattern is when the image URL contains query parameters that encode session data or secrets:
![x](https://tracker.example.com/px?uid=12345&token=abc)  ← suspicious (data in query string)

Sieve's IN-GEN-04 rule specifically targets the second pattern: Markdown images with URLs that include query parameters, which could be used to exfiltrate data to an attacker-controlled server when the image renders.
