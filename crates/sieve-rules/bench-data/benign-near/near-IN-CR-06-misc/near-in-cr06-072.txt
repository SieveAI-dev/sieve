Responsible disclosure for AI security vulnerabilities

If you discover a prompt injection vulnerability in an AI product:

1. Do not publicly disclose immediately — follow coordinated disclosure
2. Contact the vendor's security team (security@vendor.com or security.txt)
3. Provide a minimal reproducible example
4. Allow 90 days for remediation before public disclosure (standard industry practice)
5. Do not exploit in production systems or access other users' data

For Sieve: rule engine or proxy issues go to GitHub Security Advisories (private). Anthropic API issues go to security@anthropic.com.

No formal bug bounty yet — recognition in CHANGELOG and security advisories.
