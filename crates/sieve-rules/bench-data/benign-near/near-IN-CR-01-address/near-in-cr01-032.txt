GitHub issue: address comparison fails due to mixed case

**Problem**: Our frontend stores addresses in lowercase from the API response, but the smart contract emits addresses in checksummed form. `address1 === address2` returns false even though they're the same wallet.

**Root cause**: `0xab5801a7d398351b8be11c439e05c5b3259aec9b` (lowercase) !== `0xAb5801a7D398351b8bE11C439e05C5B3259aeC9B` (EIP-55) in JavaScript string comparison.

**Fix**: Always normalize before comparing:
```js
import { getAddress } from "ethers";
const normalize = (a) => getAddress(a.toLowerCase());
normalize(api_addr) === normalize(contract_addr); // true
```

**Lesson**: Never compare raw address strings. Always normalize to EIP-55 first.
