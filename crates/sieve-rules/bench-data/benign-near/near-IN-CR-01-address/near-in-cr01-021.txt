Understanding the EIP-55 checksum algorithm:

Input address (lowercase hex): 0xfb6916095ca1df60bb79ce92ce3ea74c37c5d359
Keccak256 of the hex string (without 0x):
  hash = 0x5f8a88c...  (internal computation)

For each hex character:
- If the corresponding nibble in hash >= 8, uppercase it
- Otherwise, keep lowercase

Result (EIP-55): 0xfB6916095ca1df60bB79Ce92cE3Ea74c37c5d359

This same address also appears in the EIP-55 specification itself as a test vector.
Reference: https://eips.ethereum.org/EIPS/eip-55
