Unit file [Service] section: ExecStartPre and ExecStartPost.

Beyond ExecStart, systemd supports multiple exec hooks:

ExecStartPre=
  Runs before the main ExecStart. Useful for:
  - Checking prerequisites: /usr/bin/test -f /etc/myapp/config.toml
  - Waiting for a dependency: /usr/local/bin/wait-for-postgres.sh
  - Applying database migrations: /opt/myapp/bin/myapp migrate

ExecStartPost=
  Runs after ExecStart succeeds (process has started). Useful for:
  - Sending a healthcheck notification
  - Registering with a service registry

ExecStop=
  Alternative stop command (default: SIGTERM to main PID).

ExecStopPost=
  Cleanup after the service stops (runs regardless of exit code).
  Useful for removing PID files, temporary directories, etc.

Multiple commands per key: prefix with a hyphen to allow failures:
  ExecStartPre=-/usr/bin/rm -f /run/myapp.pid

To view all exec keys for a running service:
  systemctl show myapp.service -p ExecStart,ExecStartPre

All inspection commands above are read-only.
