How launchd handles job failures and throttling.

When a launchd-managed job exits unexpectedly, launchd's behavior depends on the plist configuration:

KeepAlive = true:
  launchd restarts the job after ThrottleInterval seconds (default: 10s).
  If the job exits immediately every time, launchd backs off to avoid a restart storm.

KeepAlive = false (or omitted):
  launchd runs the job once. If it exits, the job is done until the next scheduled trigger.

To diagnose a failing launch agent:
1. Check last exit status: `launchctl list com.example.myapp`
   Look for: LastExitStatus = <non-zero value>
2. Check StandardErrorPath log (if configured in the plist)
3. Run the ProgramArguments manually in a terminal to see error output directly

Common exit codes:
  2   = usage error (wrong arguments)
  127 = executable not found (check ProgramArguments path)
  1   = generic error (check the application's own logs)

All diagnosis steps above are read-only. No service state is modified.
