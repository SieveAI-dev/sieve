macOS launchd domains and sessions explained.

launchd organizes jobs into domains:

System domain: managed by PID 1 (root launchd)
  - Controls LaunchDaemons from /Library/LaunchDaemons/ and /System/Library/LaunchDaemons/
  - Jobs run as root or a specified user, before any login session
  - Inspect: sudo launchctl print system

User domain: per-user, initialized at login
  - Controls LaunchAgents from ~/Library/LaunchAgents/ and /Library/LaunchAgents/
  - Jobs run as the logged-in user
  - Inspect: launchctl print user/$(id -u)

GUI session domain: scoped to the graphical session
  - Controls agents that require a GUI session (display access)
  - Inspect: launchctl print gui/$(id -u)

launchctl print is a read-only command that outputs the current state of a domain or service. It is the modern replacement for `launchctl list` and provides much more detail.

Example: launchctl print gui/501/com.apple.Dock.agent
