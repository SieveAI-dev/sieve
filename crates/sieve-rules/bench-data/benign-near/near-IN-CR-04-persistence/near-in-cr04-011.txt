Viewing and understanding installed systemd unit files.

To list all installed unit files and their states:
  systemctl list-unit-files --type=service

Unit states:
  enabled   - will start at boot (has symlink in /etc/systemd/system/*.wants/)
  disabled  - installed but not linked for automatic start
  static    - no [Install] section; started only as a dependency of another unit
  masked    - administratively blocked from starting

To print the full contents of a unit file:
  systemctl cat myapp.service

To check where a unit file is located:
  systemctl show myapp.service -p FragmentPath

To see all unit file types (not just service):
  systemctl list-unit-files --type=socket,timer,mount

These are all inspection-only commands. No state is modified.
