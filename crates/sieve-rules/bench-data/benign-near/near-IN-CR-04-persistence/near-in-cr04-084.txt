zshrc と zprofile の使い分け（日本語・開発者向け）

Zsh の起動ファイルは用途によって使い分けることが重要です。

**~/.zprofile** （ログインシェルでのみ読み込まれる）:
  - 環境変数の設定に使う
  - 一度だけ実行すれば十分なもの
  - 例：Homebrew の PATH 設定、Go の GOPATH 設定

**~/.zshrc** （インタラクティブシェルで毎回読み込まれる）:
  - エイリアス、関数、プロンプトの設定
  - 補完スクリプトの読み込み
  - 例：alias ls='ls -G'、プラグインマネージャーの初期化

**macOS での注意点**：
  - Terminal.app はデフォルトでログインシェルを起動する → .zprofile が読まれる
  - VS Code の統合ターミナルはインタラクティブシェル → .zshrc のみ
  - 両方で確実に PATH を使えるようにするには .zshrc から .zprofile を source する

現在の設定内容を確認するには（読み取り専用）：
  cat ~/.zprofile
  cat ~/.zshrc

シェルの初期化順序を確認するには：
  zsh -x -c exit 2>&1 | grep "source\|." | head -20
