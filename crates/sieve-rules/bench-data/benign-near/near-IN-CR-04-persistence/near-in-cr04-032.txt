Fish shell abbreviations vs aliases: which to use?

**Aliases** in fish are just wrapper functions. When you type an alias and press Enter, the alias name appears in your command history.

**Abbreviations** expand inline as you type, so your history shows the full expanded command. This makes history search much more useful.

Defining abbreviations (persisted as universal variables automatically):
  abbr --add dc 'docker compose'
  abbr --add gco 'git checkout'
  abbr --add kgp 'kubectl get pods'

Listing all abbreviations:
  abbr --list

Removing an abbreviation:
  abbr --erase dc

These commands modify universal variable storage (~/.config/fish/fish_variables), not config.fish. If you want abbreviations defined in config.fish for portability or version control, you can add `abbr --add` lines there — but that's a matter of team preference, not security.

For organization-wide fish configs, consider using a shared fish plugin (e.g., via Fisher or Oh My Fish) stored in a Git repository.
