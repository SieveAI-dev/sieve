## Configuring shell environment for Rust development

A typical .zshrc or .bashrc for a Rust developer:

```bash
# Cargo binaries (added by rustup installer to ~/.cargo/env)
. "$HOME/.cargo/env"

# Rust-related aliases
alias cb='cargo build'
alias ct='cargo test'
alias cr='cargo run'
alias cc='cargo check'
alias cf='cargo fmt'
alias ccl='cargo clippy'

# Cross-compilation helpers
alias build-linux='cargo build --target x86_64-unknown-linux-gnu'
alias build-macos='cargo build --target aarch64-apple-darwin'

# Quick benchmark with criterion
bench() {
  cargo bench -p "${1:-}" -- "${2:-}"
}
```

The `. "$HOME/.cargo/env"` line is added automatically by the rustup installer and simply sets CARGO_HOME and adds ~/.cargo/bin to PATH. It is a read-only source operation.

For a cleaner setup, some developers add the cargo env source only in .bash_profile or .zprofile (login shells), and put only aliases and functions in .bashrc / .zshrc.

Check current cargo installation: `cargo --version`, `rustup show`
