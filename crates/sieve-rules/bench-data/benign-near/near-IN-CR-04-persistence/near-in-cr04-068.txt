Understanding zsh plugin managers: oh-my-zsh vs zinit vs sheldon.

Plugin managers load zsh plugins and themes. They all modify your shell environment in memory when .zshrc is sourced, not by writing files during normal use.

oh-my-zsh:
  - Most popular, largest plugin ecosystem
  - Slower startup (loads many files unconditionally)
  - Configured via ZSH_THEME and plugins=(...) arrays in .zshrc
  - Source: source $ZSH/oh-my-zsh.sh

zinit:
  - Lazy loading: plugins load on first use
  - Faster startup than oh-my-zsh
  - Configured with zinit directives in .zshrc
  - Plugin snippets cached in ~/.local/share/zinit/

sheldon:
  - Written in Rust, TOML-based config in ~/.config/sheldon/plugins.toml
  - Generates a shell script at build time, very fast startup
  - Init in .zshrc: eval "$(sheldon source)"

None of these frameworks write to system locations during normal operation. Plugins are installed in user home directories (~/.oh-my-zsh, ~/.local/share/zinit, etc.).

To audit what's installed: ls ~/.oh-my-zsh/plugins/
