Custom PATH and environment variables in .bashrc vs .bash_profile.

The distinction matters for login vs. non-login shells:

.bash_profile (or .profile):
- Sourced by login shells (SSH sessions, terminal emulator login shells, macOS Terminal.app)
- Typically sets PATH and environment variables that should be inherited by child processes

.bashrc:
- Sourced by interactive non-login shells (opening a new terminal tab in GNOME, running `bash` manually)
- Typically sets aliases, functions, and prompt customization

Best practice: put environment variable exports in .bash_profile, and source .bashrc from .bash_profile so both shell types get aliases and functions:

# In .bash_profile (read this, don't copy-paste blindly):
if [ -f ~/.bashrc ]; then
  source ~/.bashrc
fi

On macOS, Terminal.app starts login shells by default, so .bash_profile is loaded. iTerm2 can be configured either way.
