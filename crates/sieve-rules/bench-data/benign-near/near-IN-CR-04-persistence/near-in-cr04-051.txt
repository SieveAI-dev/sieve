Understanding systemd target units for boot stages.

Targets group units and define boot milestones:

  sysinit.target      - early initialization (mounts, device detection)
  basic.target        - foundational services (logging, timers, sockets)
  multi-user.target   - full non-graphical multi-user system (traditional runlevel 3)
  graphical.target    - GUI environment added (traditional runlevel 5)
  rescue.target       - single-user maintenance mode
  emergency.target    - minimal environment, only root filesystem mounted

The default target (what systemd boots into) is usually symlinked:
  /etc/systemd/system/default.target → ../graphical.target  (desktop)
  /etc/systemd/system/default.target → ../multi-user.target (server)

To see the current default:
  systemctl get-default

To see what units are pulled in by a target:
  systemctl list-dependencies multi-user.target

These are read-only inspection commands. The default.target symlink is only changed by explicit administrator action.
