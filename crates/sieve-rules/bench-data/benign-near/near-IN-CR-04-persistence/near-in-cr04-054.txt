DevOps: systemd unit file for a Rust web application.

When deploying a Rust binary as a system service, a typical unit file looks like this (for review and understanding):

[Unit]
Description=My Rust API Server
Documentation=https://github.com/example/myapi
After=network-online.target postgresql.service
Wants=network-online.target

[Service]
Type=notify
User=myapi
Group=myapi
WorkingDirectory=/opt/myapi
ExecStart=/opt/myapi/bin/myapi --config /etc/myapi/config.toml
Restart=on-failure
RestartSec=5s
# Security hardening
NoNewPrivileges=true
PrivateTmp=true
ProtectSystem=strict
ReadWritePaths=/var/lib/myapi

[Install]
WantedBy=multi-user.target

Key security options:
  NoNewPrivileges   - prevents privilege escalation via setuid binaries
  PrivateTmp        - gives the service a private /tmp namespace
  ProtectSystem     - makes /usr and /boot read-only to the service

To validate this file: systemd-analyze verify myapi.service
