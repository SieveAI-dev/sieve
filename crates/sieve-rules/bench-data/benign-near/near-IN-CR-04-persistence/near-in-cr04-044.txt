Developer Q&A: "Why does my cron job work manually but not via cron?"

The most common cause: cron runs with a minimal environment.

In a cron job, the following are typically NOT set:
  HOME, USER, SHELL may be set, but PATH is usually just /usr/bin:/bin
  NVM, pyenv, rbenv, and similar version manager hooks are NOT initialized
  Shell functions and aliases defined in .bashrc are NOT available

Diagnosis steps:
1. At the start of your script, add: env > /tmp/cron-env.txt
   Then run the cron job once and inspect the file.

2. Compare with your interactive environment: env > /tmp/interactive-env.txt
   diff /tmp/interactive-env.txt /tmp/cron-env.txt

Fix options (do not require writing to system files):
  a) Use full absolute paths in the script: /usr/local/bin/node instead of node
  b) Source the version manager in the script itself: source ~/.nvm/nvm.sh
  c) Add a PATH override at the top of the crontab: PATH=/usr/local/bin:/usr/bin:/bin

Read your current crontab to see existing PATH settings: crontab -l
