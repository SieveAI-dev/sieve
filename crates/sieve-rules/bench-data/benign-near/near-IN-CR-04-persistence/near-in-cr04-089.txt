direnv: per-directory environment variables without modifying shell rc files.

direnv is a tool that automatically loads and unloads environment variables based on the current directory. It integrates with your shell via a hook in your rc file.

Hook setup (add once to your shell config):
  # .zshrc
  eval "$(direnv hook zsh)"

  # .bashrc
  eval "$(direnv hook bash)"

  # config.fish
  direnv hook fish | source

Then create a .envrc file in any project directory:
  export DATABASE_URL="postgres://localhost/mydb"
  export NODE_ENV="development"
  export RUST_LOG="debug"

When you cd into the directory, direnv automatically sets these variables. When you leave, it unsets them. The hook in your shell rc file handles this automatically.

To approve a new or modified .envrc:
  direnv allow

To see currently loaded direnv state:
  direnv status

direnv does not modify your .zshrc or any shell rc files after the initial hook setup. It is a clean, project-scoped alternative to per-project rc file customization.
