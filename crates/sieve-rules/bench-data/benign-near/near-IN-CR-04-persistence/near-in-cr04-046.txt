systemd-analyze: diagnosing slow boot times.

systemd-analyze provides tools for understanding boot performance.

Total boot time breakdown:
  systemd-analyze

Example output:
  Startup finished in 1.234s (firmware) + 2.345s (loader) + 3.456s (kernel) + 8.123s (userspace) = 15.158s
  graphical.target reached after 8.100s in userspace.

Show which units consumed the most time:
  systemd-analyze blame

Visualize the critical dependency chain:
  systemd-analyze critical-chain graphical.target

Generate an interactive SVG boot chart (opens in browser):
  systemd-analyze plot > /tmp/bootchart.svg

Check for unit file errors without applying changes:
  systemd-analyze verify /path/to/myapp.service

The verify subcommand is particularly useful for validating a unit file before placing it in the correct location. It catches syntax errors and references to missing units.

All of these are read-only analysis operations.
