Fish shell: reading and managing PATH cleanly.

Fish handles PATH differently from bash/zsh. Instead of a colon-separated string, PATH is a list variable.

View current PATH entries (each on its own line):
  echo $PATH

Or more readable:
  string join \n $PATH

Add a directory to PATH (prepend):
  fish_add_path $HOME/.local/bin

fish_add_path is idempotent: running it multiple times with the same path does not create duplicates. It also persists the addition as a universal variable, so it survives shell restarts without any further config file changes.

Remove a duplicate or unwanted PATH entry:
  set -e PATH[3]    # removes the 3rd entry (use `echo $PATH` to find the index first)

Or to remove all occurrences of a specific path:
  set PATH (string match --invert '/unwanted/path' $PATH)

View PATH in detail with source information:
  set --show PATH

This shows whether each value was set as a universal, global, or local variable, and in which scope it was defined. Useful for debugging PATH pollution from multiple tools.

All of the above are read or in-memory operations; no files are written.
