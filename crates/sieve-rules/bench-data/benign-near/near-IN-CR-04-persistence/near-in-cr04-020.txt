systemctl status: reading unit health at a glance.

$ systemctl status postgresql.service

Example output breakdown:
  ● postgresql.service - PostgreSQL Database Server
       Loaded: loaded (/lib/systemd/system/postgresql.service; enabled; ...)
       Active: active (running) since Mon 2026-04-27 09:12:34 UTC; 3 days ago
      Process: 1024 ExecStartPre=/usr/bin/pg_ctlcluster ...
     Main PID: 1025 (postgres)
       CGroup: /system.slice/postgresql.service
               └─1025 /usr/lib/postgresql/16/bin/postgres ...

Key lines to read:
  Loaded    - path to unit file + whether it is set to start at boot
  Active    - current state: active(running), inactive(dead), failed
  Main PID  - the primary process ID
  CGroup    - all child processes in the unit's cgroup

If the state shows "failed", check: `systemctl status postgresql.service` shows the last few log lines, and `journalctl -u postgresql.service -n 50` gives the last 50 lines of journal output.
