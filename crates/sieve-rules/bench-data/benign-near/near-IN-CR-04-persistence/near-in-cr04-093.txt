Understanding macOS LaunchAgent plist KeepAlive dictionary.

A simple `KeepAlive = true` in a plist makes launchd restart the job unconditionally. But the KeepAlive key also accepts a dictionary for fine-grained control.

KeepAlive dictionary keys:

SuccessfulExit (boolean):
  true  → restart only if the job exits with code 0
  false → restart only if the job exits with a non-zero code

Crashed (boolean):
  true  → restart only if the job was killed by a signal (crashed or killed)

PathState (dictionary):
  Maps filesystem paths to booleans.
  true  → keep alive while the path exists
  false → keep alive while the path does NOT exist

Example: keep a sync daemon alive only while a mounted volume exists:

KeepAlive:
  PathState:
    /Volumes/ExternalDrive: true

OtherJobEnabled (dictionary):
  Maps job labels to booleans.
  Keeps this job alive only while another named job is enabled.

NetworkState (boolean):
  true  → keep alive only when the network is available

Inspect KeepAlive settings for a loaded job:
  launchctl list com.example.myapp
  plutil -p ~/Library/LaunchAgents/com.example.myapp.plist
