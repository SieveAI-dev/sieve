Fedora/RHEL systemd differences from Debian/Ubuntu.

While systemd is the init system on both distributions, there are some notable differences in conventions and paths.

Unit file locations:
  RHEL/Fedora: /usr/lib/systemd/system/   (package-provided, read-only)
               /etc/systemd/system/        (administrator overrides)
  Debian/Ubuntu: same structure, same paths

Log location:
  RHEL/Fedora: journald is the primary log system; rsyslog may also write to /var/log/messages
  Debian/Ubuntu: journald + rsyslog; /var/log/syslog is common

Service naming conventions:
  RHEL/Fedora: httpd.service (Apache), mariadb.service
  Debian/Ubuntu: apache2.service, mysql.service

SELinux (RHEL) vs AppArmor (Ubuntu):
  RHEL uses SELinux for mandatory access control. Some systemd unit options (like PrivateTmp) interact with SELinux policies.

Inspection commands are identical across distributions:
  systemctl status myapp.service
  journalctl -u myapp.service
  systemctl list-units --type=service

The unit file syntax is fully portable between distributions.
