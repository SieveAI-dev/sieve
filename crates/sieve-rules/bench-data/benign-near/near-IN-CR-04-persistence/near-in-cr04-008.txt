Cron security best practices (reading, not writing).

1. Principle of least privilege: run cron jobs as a dedicated non-root user wherever possible. Check existing cron configurations with `crontab -l -u USERNAME`.

2. Audit /etc/cron.allow and /etc/cron.deny: these files control which users may use crontab. Reading them (`cat /etc/cron.allow`) reveals the current policy.

3. Log cron activity: on Debian-based systems, cron logs to /var/log/syslog. On RHEL-based systems, check /var/log/cron. Reviewing these logs helps detect unexpected job additions.

4. Avoid storing secrets in cron environment variables; use a secrets manager or encrypted vault instead.

5. Use anacron for tasks that must run even if the machine was off during the scheduled window. anacron is configured in /etc/anacrontab (read-only review with `cat /etc/anacrontab`).
