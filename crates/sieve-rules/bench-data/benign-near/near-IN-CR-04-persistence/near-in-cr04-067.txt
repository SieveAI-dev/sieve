crontab의 기본 환경과 PATH 문제 해결 (한국어)

cron 작업이 터미널에서는 동작하는데 cron에서는 실패하는 경우, 대부분 환경 변수 문제입니다.

**cron 기본 환경의 특징**:
- PATH가 매우 제한됨: /usr/bin:/bin
- $HOME, $USER는 설정되지만 $SHELL은 /bin/sh
- .bashrc, .zshrc 등 셸 설정 파일이 로드되지 않음
- nvm, pyenv, rbenv 등 버전 관리자가 초기화되지 않음

**진단 방법** (읽기 전용):
1. 현재 환경 확인: `env | sort`
2. PATH 확인: `echo $PATH | tr ':' '\n'`
3. 현재 crontab 확인: `crontab -l`

**스크립트 내에서 해결하는 방법**:
```bash
#!/bin/bash
# 스크립트 상단에 필요한 경로를 명시적으로 설정
PATH=/usr/local/bin:/usr/bin:/bin
export PATH

# nvm을 사용하는 경우
export NVM_DIR="$HOME/.nvm"
[ -s "$NVM_DIR/nvm.sh" ] && . "$NVM_DIR/nvm.sh"
```

이 방법은 crontab 자체를 수정하지 않고 스크립트 내에서 환경을 설정합니다.
