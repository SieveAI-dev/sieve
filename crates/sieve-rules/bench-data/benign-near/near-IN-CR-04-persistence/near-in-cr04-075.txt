mise (formerly rtx): managing language runtimes without modifying system files.

mise is a tool for managing multiple versions of language runtimes (Node.js, Python, Ruby, Go, Rust, etc.). It reads a .tool-versions file in the project directory.

Activate mise in your shell (one line added to shell config):
  eval "$(mise activate zsh)"    # in .zshrc
  eval "$(mise activate bash)"   # in .bashrc
  mise activate fish | source    # in config.fish

After activation, mise intercepts PATH lookups and routes to the correct runtime version based on the nearest .tool-versions file (or ~/.tool-versions for global defaults).

List installed runtimes:
  mise list

Show current runtime versions in the active directory:
  mise current

Show which .tool-versions file is controlling a directory:
  mise where node

mise does not modify system directories. All runtimes are installed under ~/.local/share/mise/installs/. No sudo required, no system PATH modification.

This is the recommended pattern for project-level runtime pinning without touching shell rc files beyond the initial activation line.
