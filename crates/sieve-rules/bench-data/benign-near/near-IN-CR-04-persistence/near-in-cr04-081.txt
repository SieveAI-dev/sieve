Listing and inspecting /etc/cron.daily/ scripts.

The directories /etc/cron.daily/, /etc/cron.weekly/, and /etc/cron.monthly/ contain executable scripts (not crontab-format files) that are run by the system cron daemon at the appropriate interval.

To see what scripts are installed:
  ls -la /etc/cron.daily/
  ls -la /etc/cron.weekly/
  ls -la /etc/cron.monthly/

To read a specific script (example — logrotate):
  cat /etc/cron.daily/logrotate

Typical contents of /etc/cron.daily/:
  logrotate       - rotates log files (reads /etc/logrotate.conf)
  apt-compat      - Debian/Ubuntu package cleanup
  man-db          - updates the man page database
  mlocate or plocate - updates the locate file index

On systemd-based systems, these directories may be handled by run-parts in the system crontab (/etc/crontab) or via systemd timer units:
  systemctl list-timers | grep daily

All of the above are read-only inspection operations.
