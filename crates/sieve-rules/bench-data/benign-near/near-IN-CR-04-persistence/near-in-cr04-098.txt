Systemd unit templates: parameterized services.

A template unit has an @ in the filename: myapp@.service
It can be instantiated with different parameters: myapp@8080.service, myapp@8081.service

Inside the template, use %i to refer to the instance identifier:

[Unit]
Description=My App instance %i

[Service]
ExecStart=/usr/local/bin/myapp --port %i
User=myapp

[Install]
WantedBy=multi-user.target

To check all instances of a template:
  systemctl list-units 'myapp@*'

To see the status of a specific instance:
  systemctl status myapp@8080.service

Common use case: running multiple instances of a service on different ports, or per-user service instances.

Example: SSH service uses a template — each SSH connection spawns:
  sshd@192.168.1.5:22-192.168.1.100:54321.service

Inspect running SSH connection units:
  systemctl list-units 'sshd@*'

All of these are read-only inspection commands.
