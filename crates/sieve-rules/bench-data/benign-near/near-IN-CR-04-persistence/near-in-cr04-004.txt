Customizing your .zshrc for a better developer experience.

Common things users add to their .zshrc:

# Set a custom PATH so local scripts are found first
export PATH="$HOME/.local/bin:$PATH"

# Aliases for frequent commands
alias gs='git status'
alias gl='git log --oneline -20'
alias ll='ls -la'

# A handy function for creating and entering a directory
mkcd() {
  mkdir -p "$1" && cd "$1"
}

# Load a version manager (example: mise)
eval "$(mise activate zsh)"

To apply changes without opening a new shell, run: source ~/.zshrc
Note: source is a read operation only; it does not modify the file.
