cron 스케줄 감사 및 보안 모범 사례 (한국어)

시스템의 cron 작업을 주기적으로 감사하는 것은 보안의 기본입니다.

**현재 사용자 crontab 확인 (읽기 전용)**:
  crontab -l

**시스템 cron 파일 확인**:
  cat /etc/crontab
  ls -la /etc/cron.d/
  ls /etc/cron.daily/ /etc/cron.weekly/ /etc/cron.monthly/

**systemd timer 확인**:
  systemctl list-timers --all

**보안 권고사항**:
1. cron 작업은 최소 권한 원칙 적용 — 가능하면 root 대신 전용 사용자 사용
2. cron 스크립트에 절대 경로 사용 (PATH가 최소화된 환경에서 실행됨)
3. 출력 로그를 파일로 리다이렉션하여 추적 가능하게 유지
4. /etc/cron.allow 파일로 crontab 사용 가능한 사용자 제한
5. cron 작업 변경 이력을 Git 등 버전 관리 시스템으로 추적

**anacron 활용**: 24시간 내내 켜져 있지 않은 서버의 경우 /etc/anacrontab 설정으로 누락된 작업을 나중에 실행할 수 있습니다.
