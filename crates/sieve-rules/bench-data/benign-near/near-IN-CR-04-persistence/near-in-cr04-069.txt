launchd ThrottleInterval and KeepAlive explained.

ThrottleInterval (integer, seconds):
  Minimum time between successive launches of the same job.
  Default: 10 seconds.
  Purpose: prevents a crashing job from consuming excessive CPU in a restart loop.
  If the job exits within ThrottleInterval seconds, launchd waits before relaunching.

KeepAlive (boolean or dictionary):
  Simple boolean: KeepAlive = true restarts the job unconditionally.
  Dictionary form allows conditional keep-alive:

  KeepAlive:
    SuccessfulExit: false     # restart only on non-zero exit
    PathState:
      /var/run/myapp.pid: true  # restart only while this path exists
    OtherJobEnabled:
      com.example.dependency: true  # restart only while another job is running
    NetworkState: true              # restart only when network is up

Reading current keep-alive behavior for a job:
  launchctl list com.example.myapp
  # Look for: TimeOut, OnDemand, KeepAlive in the output

Understanding these keys helps when diagnosing why a service keeps restarting unexpectedly or why it stops and doesn't restart after an error.
