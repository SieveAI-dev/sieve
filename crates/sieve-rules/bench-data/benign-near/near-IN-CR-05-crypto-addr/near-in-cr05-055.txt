Solana address format: Base58 without the checksum

Solana public keys are 32-byte Ed25519 public keys encoded in Base58 (Bitcoin alphabet).
Unlike Bitcoin's Base58Check, there is no appended checksum.

Characteristics:
- Always 32 bytes (256 bits), encoded to 43-44 characters
- Uses the Bitcoin Base58 alphabet: 123456789ABCDEFGHJKLMNPQRSTUVWXYZabcdefghijkmnopqrstuvwxyz
- No "0", "O", "I", "l" to avoid visual confusion
- On-chain validation: the runtime checks the point is on the Ed25519 curve

Example: vines1vzrYbzLMRdu58ou5XTby4qAqVRLmqo36NKPTg (43 chars)
Invalid: 0OI (these chars never appear in a valid Solana address)
