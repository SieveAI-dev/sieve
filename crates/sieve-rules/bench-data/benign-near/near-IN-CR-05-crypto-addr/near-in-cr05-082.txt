ENS / SNS name squatting and resolution spoofing:

ENS security note: ENS names with lookalike Unicode characters can fool users.
Example: "vïtalik.eth" (with ï, U+00EF) vs "vitalik.eth" — visually identical in some fonts.

IDNA (Internationalized Domain Names) normalization applies to ENS:
- The ENS app normalizes names before displaying
- Always verify the resolved address, not just the displayed name:
  await provider.resolveName("vïtalik.eth")  // resolves to attacker's address

Mitigation:
- Wallets should display both the ENS name AND the resolved EVM address
- For high-value transfers, verify address independently (e.g., Etherscan lookup)
- Address: 0xd8dA6BF26964aF9D7eEd9e03E53415D37aA96045 = official vitalik.eth
