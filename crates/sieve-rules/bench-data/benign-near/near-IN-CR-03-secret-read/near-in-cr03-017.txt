ProxyJump / ProxyCommand for bastion hosts

Modern approach (OpenSSH 7.3+) uses ProxyJump:

    Host internal-server
        HostName 10.10.0.5
        User ubuntu
        ProxyJump bastion.example.com

Older approach uses ProxyCommand:

    Host internal-server
        HostName 10.10.0.5
        User ubuntu
        ProxyCommand ssh -W %h:%p bastion.example.com

Both approaches tunnel through the bastion transparently. Your SSH client
handles the two-hop authentication; the bastion host never sees your private
key if you use `ForwardAgent no` (the default and recommended setting).

For strict security, use `ForwardAgent yes` only on trusted bastions and
prefer `-J` (ProxyJump) which does not enable agent forwarding by default.
