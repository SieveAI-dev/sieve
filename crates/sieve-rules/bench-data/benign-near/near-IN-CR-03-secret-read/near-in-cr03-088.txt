Rclone and .netrc alternative

Rclone (cloud storage CLI) uses its own config file at
`~/.config/rclone/rclone.conf` rather than .netrc. It supports encryption:

    # Create an encrypted rclone config (prompts for config password)
    rclone config

    # Store the config password in the environment for CI:
    RCLONE_CONFIG_PASS=mypassword rclone copy ...

Rclone.conf format (example with S3 — using environment vars, not hardcoded):
    [mys3]
    type = s3
    provider = AWS
    access_key_id = ${AWS_ACCESS_KEY_ID}
    secret_access_key = ${AWS_SECRET_ACCESS_KEY}
    region = us-east-1

The `${VAR}` interpolation means credentials come from the environment
at runtime, not stored in the config file. For CI, set AWS_ACCESS_KEY_ID
and AWS_SECRET_ACCESS_KEY via platform secrets.

Set permissions: chmod 600 ~/.config/rclone/rclone.conf
