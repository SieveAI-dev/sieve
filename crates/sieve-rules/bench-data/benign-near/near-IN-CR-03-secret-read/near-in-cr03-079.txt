GPG agent configuration

The `gpg-agent` caches passphrases so you don't re-enter them constantly.
Configure via `~/.gnupg/gpg-agent.conf`:

    # Cache passphrase for 1 hour (in seconds)
    default-cache-ttl 3600
    max-cache-ttl 7200

    # macOS: use the Pinentry from macOS dialog (nicer UI)
    pinentry-program /usr/local/bin/pinentry-mac

    # For SSH authentication via GPG agent:
    enable-ssh-support

    # Restart the agent after config changes:
    gpgconf --kill gpg-agent

Using GPG agent for SSH (replaces ssh-agent):
    # Add to shell profile:
    export SSH_AUTH_SOCK=$(gpgconf --list-dirs agent-ssh-socket)

With enable-ssh-support, your GPG authentication subkey can be used for SSH
login without a separate SSH key pair. Both the SSH and GPG passphrases
are cached by the same agent.
