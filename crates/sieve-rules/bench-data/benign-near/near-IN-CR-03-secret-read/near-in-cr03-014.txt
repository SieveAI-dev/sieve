SSH key generation best practices (2025)

Recommended algorithm: Ed25519
    ssh-keygen -t ed25519 -C "your_email@example.com"

Ed25519 is faster, produces shorter keys, and is considered more secure
than RSA-2048 for new keys. For systems that require RSA:
    ssh-keygen -t rsa -b 4096 -C "your_email@example.com"

After generating:
1. Add the *public* key (`~/.ssh/id_ed25519.pub`) to the server's
   `~/.ssh/authorized_keys` or to GitHub/GitLab.
2. Never share the private key file (no `.pub` extension).
3. Always set a strong passphrase — use a password manager to store it.
4. Set correct permissions:
   chmod 700 ~/.ssh
   chmod 600 ~/.ssh/id_ed25519
   chmod 644 ~/.ssh/id_ed25519.pub
