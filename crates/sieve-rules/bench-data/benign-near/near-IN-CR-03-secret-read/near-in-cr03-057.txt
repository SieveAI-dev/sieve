Solana program deploy keypair vs. wallet keypair

Two separate keypairs matter for Solana programs:

1. Wallet keypair (~/.config/solana/id.json):
   - Pays for transaction fees during deployment
   - Holds SOL for rent-exemption deposits
   - Acts as initial upgrade authority

2. Program keypair (target/deploy/<name>-keypair.json in Anchor):
   - Determines the program's on-chain address
   - Used only during the first deploy to "claim" that address
   - After deploy, upgrade authority is what matters

Best practice for production:
    # After initial deploy, transfer upgrade authority to a multisig
    solana program set-upgrade-authority PROGRAM_ID       --new-upgrade-authority MULTISIG_ADDRESS

    # Then the program keypair is no longer needed for upgrades
    # Keep it only for disaster recovery

Store both keypairs securely; losing the program keypair doesn't prevent
upgrades if you have the upgrade authority.
