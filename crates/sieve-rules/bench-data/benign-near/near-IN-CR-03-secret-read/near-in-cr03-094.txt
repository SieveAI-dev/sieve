git-secrets by AWS Labs

git-secrets scans commits, branches, and history for secret patterns:

    # Install (macOS)
    brew install git-secrets

    # Register AWS patterns (detects AWS access keys and secret keys)
    git secrets --register-aws --global

    # Install hooks in current repo
    git secrets --install

    # Scan the entire git history
    git secrets --scan-history

    # Add a custom pattern (example: detect internal API key format)
    git secrets --add 'INTERNAL_KEY_[A-Z0-9]{32}'

    # Add a literal string to the allowlist (for test fixtures)
    git secrets --add --allowed 'EXAMPLE_KEY_123456'

If git-secrets finds a match, the commit is rejected with a clear error.
The developer must either remove the secret or (if it's a false positive)
add it to the allowlist with justification.

Pair with detect-secrets for comprehensive coverage — they use different
detection heuristics and complement each other.
