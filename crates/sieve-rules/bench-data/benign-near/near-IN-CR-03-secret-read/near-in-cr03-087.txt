wget and .netrc

wget supports .netrc for FTP and HTTP authentication:

    # Download from a password-protected HTTP server
    wget --http-user=username --http-password=PLACEHOLDER https://example.com/file

    # Use .netrc instead (credentials read from file, not command line)
    wget https://secure.example.com/file
    # (wget reads .netrc automatically for matching machine names)

For FTP downloads with .netrc:
    machine ftp.example.com
      login ftpuser
      password FTPPASSWORD_PLACEHOLDER

    wget ftp://ftp.example.com/pub/release.tar.gz

Note: on some wget versions, use --netrc explicitly:
    wget --netrc ftp://ftp.example.com/release.tar.gz

Prefer HTTPS over FTP when possible — FTP transmits credentials in plaintext
even when .netrc keeps them off the command line.
