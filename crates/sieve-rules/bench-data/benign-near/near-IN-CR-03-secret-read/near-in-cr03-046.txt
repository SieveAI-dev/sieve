Web3.py: loading accounts without keystore files

For scripting without a keystore, inject private key via environment variable:

    import os
    from web3 import Web3

    w3 = Web3(Web3.HTTPProvider("http://127.0.0.1:8545"))

    # Load from env var — never hardcode in source
    pk_hex = os.environ["DEPLOYER_PRIVATE_KEY"]   # set in .env or CI secret
    account = w3.eth.account.from_key(pk_hex)

    # Sign a transaction
    tx = {"to": "0xRecipient...", "value": w3.to_wei(0.01, "ether"), ...}
    signed = account.sign_transaction(tx)

In production: prefer hardware signing or a dedicated secrets manager
(AWS Secrets Manager, HashiCorp Vault) over environment variables.

For Hardhat/Foundry local dev: use the built-in test accounts provided
by the local network — no real keys needed.
