Writing a solid `.env.example` template:

1. List every variable your application reads.
2. Provide safe placeholder values, not real secrets.
3. Add a comment for each variable explaining where to obtain the value.

Example:

    # Database
    DATABASE_URL=postgres://user:password@localhost:5432/mydb

    # Auth (generate with: openssl rand -hex 32)
    SESSION_SECRET=REPLACE_WITH_32_RANDOM_HEX_BYTES

    # Third-party integrations (get from provider dashboard)
    STRIPE_PUBLISHABLE_KEY=pk_test_REPLACE_ME
    SENDGRID_API_KEY=SG.REPLACE_ME

Onboarding new developers: `cp .env.example .env` then fill in real values.
