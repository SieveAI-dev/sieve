Solana Anchor framework and keypair management

Anchor uses keypairs for program deployment and upgrade authority:

    # In Anchor.toml, specify the wallet:
    [provider]
    cluster = "devnet"
    wallet = "~/.config/solana/id.json"

    # Deploy a program (uses the configured wallet as payer and authority)
    anchor deploy

    # Use a custom keypair for tests
    anchor test --provider.wallet ~/solana-keys/test.json

For program IDs, Anchor auto-generates a program keypair in target/deploy/:
    target/deploy/my_program-keypair.json

This keypair determines the program address. Back it up — losing it means
you cannot upgrade the program (unless you transferred upgrade authority).

In production, transfer upgrade authority to a multisig (Squads protocol)
rather than a single hot wallet keypair.
