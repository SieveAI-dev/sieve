Using .netrc with curl

    # curl reads ~/.netrc automatically with -n / --netrc
    curl --netrc https://api.private-service.example.com/data

    # Specify an alternate netrc file location
    curl --netrc-file /path/to/my-netrc https://api.example.com/data

    # .netrc format that curl understands:
    machine api.example.com
      login myuser
      password APIKEY_OR_TOKEN

    # For token-based APIs, put the token as the password:
    machine api.github.com
      login token
      password ghp_PLACEHOLDER_REPLACE_WITH_REAL_TOKEN

Benefits over `-u user:pass` flag:
- Credentials don't appear in shell history
- Credentials don't appear in process list (ps aux)
- Can be reused across scripts without duplication

Security: curl checks file permissions and warns if .netrc is world-readable.
