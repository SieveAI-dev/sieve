SSH-agent forwarding: when and when not to use it

`ForwardAgent yes` in `~/.ssh/config` or `-A` flag allows remote servers to
use your local ssh-agent for further hops. This is convenient for multi-hop
deployments but comes with risk: a compromised intermediate server can use
your agent socket to authenticate as you to other services.

Prefer ProxyJump (see Host internal-server example in ssh config) which
routes connections without forwarding the agent.

When agent forwarding is unavoidable:
- Limit to specific hosts in ~/.ssh/config rather than setting it globally
- Use short-lived keys or certificates with expiry
- Monitor agent socket access: echo $SSH_AUTH_SOCK

Never set ForwardAgent globally in your ssh config with Host *.
