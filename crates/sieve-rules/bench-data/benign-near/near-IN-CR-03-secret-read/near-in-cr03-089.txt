Using .netrc with Docker

Docker Hub authentication uses `~/.docker/config.json`, not .netrc.
But some Docker features interact with .netrc for build-time dependencies:

    # During docker build, if you need to pull a package from a private registry:
    # Use BuildKit secrets (preferred — doesn't bake credentials into layers):
    docker build --secret id=netrc,src=$HOME/.netrc .

    # In Dockerfile:
    RUN --mount=type=secret,id=netrc,target=/root/.netrc         pip install mypackage --index-url https://private.pypi.example.com/simple/

With BuildKit secrets, the .netrc file is available only during that RUN step
and is not included in the final image or any intermediate layer.

For docker-compose builds:
    services:
      build:
        build:
          context: .
          secrets:
            - netrc
    secrets:
      netrc:
        file: ~/.netrc
