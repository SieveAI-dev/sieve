detect-secrets: preventing accidental credential commits

Installation and setup:

    pip install detect-secrets

    # Initialize baseline (scan the current repo)
    detect-secrets scan > .secrets.baseline

    # Audit the baseline (mark false positives as benign)
    detect-secrets audit .secrets.baseline

    # Add to .pre-commit-config.yaml:
    repos:
      - repo: https://github.com/Yelp/detect-secrets
        rev: v1.4.0
        hooks:
          - id: detect-secrets
            args: ['--baseline', '.secrets.baseline']

    # Install the hook
    pre-commit install

detect-secrets uses entropy analysis and pattern matching — it catches:
- High-entropy strings (likely keys)
- Common API key formats (AWS, GitHub, Stripe, etc.)
- Private key headers (PEM format)
- .env-style KEY=VALUE pairs with high-entropy values

The baseline file should be committed; it tracks known false positives
(test fixtures, example values) so they don't block every commit.
