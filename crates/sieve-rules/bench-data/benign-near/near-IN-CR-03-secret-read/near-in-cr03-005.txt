.env.local vs .env.production — common point of confusion

`.env.local`:
  - Machine-specific overrides; always git-ignored.
  - Suitable for developer-specific values (local DB port, debug flags).
  - Loaded in both development and test modes; NOT in production.

`.env.production`:
  - Environment-specific defaults for production mode.
  - May be committed if it contains only non-secret defaults (e.g.,
    LOG_LEVEL=info). Secrets should NEVER appear here — use platform secrets.
  - Only loaded when NODE_ENV=production (or VITE_MODE=production).

Precedence (highest → lowest):
  .env.[mode].local > .env.[mode] > .env.local > .env
