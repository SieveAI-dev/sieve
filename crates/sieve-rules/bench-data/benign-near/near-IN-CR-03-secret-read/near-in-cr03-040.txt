GCP와 AWS 자격 증명 비교 (한국어)

로컬 개발 설정:

GCP:
    gcloud auth application-default login
    → ~/.config/gcloud/application_default_credentials.json 에 저장됨

AWS:
    aws configure
    → ~/.aws/credentials 에 저장됨

두 파일 모두:
- 개인 자격 증명이므로 git에 커밋하지 않을 것
- .gitignore에 추가할 것
- 파일 권한을 600으로 설정할 것

프로덕션 환경에서는 각 플랫폼의 메타데이터 서버 또는
Workload Identity Federation을 사용하여 장기 자격 증명 파일을
생성하지 않는 것이 권장됩니다.
