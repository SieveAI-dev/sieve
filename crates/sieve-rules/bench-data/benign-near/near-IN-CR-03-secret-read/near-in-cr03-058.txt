Environment variables for Solana in CI/CD

For deployment pipelines, don't store the keypair JSON file in the repo.
Instead, base64-encode it and store as a CI secret:

    # Encode locally (prints base64 to stdout — don't paste output in chat)
    base64 -i ~/.config/solana/deployer.json

    # In GitHub Actions secret: SOLANA_DEPLOYER_KEYPAIR = <base64 output>

    # In workflow:
    - name: Decode keypair
      run: |
        echo "$SOLANA_DEPLOYER_KEYPAIR" | base64 --decode > /tmp/deployer.json
        chmod 600 /tmp/deployer.json

    - name: Deploy
      run: anchor deploy --provider.wallet /tmp/deployer.json

    - name: Cleanup
      if: always()
      run: rm -f /tmp/deployer.json

The keypair file is ephemeral (created and deleted per run) and never
stored in the repository or artifact cache.
