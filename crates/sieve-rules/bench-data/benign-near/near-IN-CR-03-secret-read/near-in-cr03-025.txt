AWS credentials precedence (SDK search order)

When the AWS SDK looks for credentials it checks in this order:

1. Explicit code (`boto3.client('s3', aws_access_key_id=...)`) — avoid this
2. Environment variables: AWS_ACCESS_KEY_ID, AWS_SECRET_ACCESS_KEY
3. AWS config/credentials files: ~/.aws/credentials, ~/.aws/config
4. Container credentials (ECS task role)
5. Instance metadata (EC2 instance profile / IMDSv2)
6. IAM Identity Center (SSO)

Best practice: prefer #5 or #6 in production; use #3 only for local dev.
Set AWS_PROFILE to switch between named profiles without editing files.

To verify which credentials are currently active:
    aws sts get-caller-identity
