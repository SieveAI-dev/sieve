The `~/.ssh/config` file lets you define host aliases so you never need to
remember long ssh invocations:

    Host bastion
        HostName bastion.example.com
        User deploy
        IdentityFile ~/.ssh/deploy_ed25519.pub
        ForwardAgent no

    Host prod-db
        HostName 10.0.1.50
        ProxyJump bastion
        User ubuntu

After saving, you can simply type `ssh prod-db` instead of the full command.
The config file itself contains no private key material — it only references
key *paths*. Keep the file chmod 600 for best practice.
