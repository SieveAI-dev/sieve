HD wallets and the keystore format

BIP-32/39/44 hierarchical deterministic wallets derive all account keys
from a single mnemonic seed phrase. This is fundamentally different from
the keystore format:

Keystore (geth):
  - One file = one encrypted private key
  - Backup requires keeping the file + its passphrase
  - Default path: ~/.ethereum/keystore/

HD wallet (MetaMask, Ledger):
  - One mnemonic → unlimited accounts via derivation path
  - Backup is the mnemonic phrase (12 or 24 BIP-39 words)
  - No per-account keystore file needed

For smart contract development (Foundry / Hardhat):
    # Use an encrypted HD wallet keystore with Foundry's cast wallet
    cast wallet new --keystore-dir ./keystore
    # This creates a geth-compatible keystore file in ./keystore/

Never store mnemonic phrases in `.env` files in production code.
