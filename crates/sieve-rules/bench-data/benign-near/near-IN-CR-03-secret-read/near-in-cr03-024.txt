IAM roles as a replacement for long-term credentials

The `~/.aws/credentials` file is a local convenience for human developers.
For production workloads, use IAM roles instead:

EC2 instance:
  Attach an instance profile → the SDK finds credentials via IMDS automatically.
  No credentials file needed on the instance at all.

Lambda:
  Execution role is assigned in the function configuration.

ECS / Fargate:
  Task role is defined in the task definition.

On-premise or GitHub Actions:
  Use OIDC federation → exchange a short-lived OIDC token for AWS credentials.
  Example GitHub Actions:
    - uses: aws-actions/configure-aws-credentials@v4
      with:
        role-to-assume: arn:aws:iam::ACCOUNT_ID:role/GitHubActionsRole
        aws-region: us-east-1

With OIDC, no long-term secrets are stored anywhere in the pipeline.
