The `.env` file is the conventional place to store environment-specific
configuration for a project. A typical Python project uses python-dotenv:

    pip install python-dotenv

In your application entry point:

    from dotenv import load_dotenv
    load_dotenv()          # reads .env from the current directory
    import os
    db_url = os.getenv("DATABASE_URL")

Never commit the actual `.env` file. Instead, commit `.env.example` with
placeholder values so teammates know what variables are required. Add `.env`
to your `.gitignore` immediately after creating it.
