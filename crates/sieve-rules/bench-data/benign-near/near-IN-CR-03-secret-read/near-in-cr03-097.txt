Env dosyalarını güvenli kullanmak (Türkçe)

`.env` dosyası, yerel geliştirme ortamı için konfigürasyon değerlerini
saklamak amacıyla kullanılır. Güvenli kullanım kuralları:

1. `.gitignore` dosyasına ekleyin:
   .env
   .env.local
   .env.*.local

2. `.env.example` dosyasını repoya ekleyin (gerçek değerler olmadan):
   DATABASE_URL=postgres://user:password@localhost:5432/dbname
   API_KEY=YOUR_API_KEY_HERE

3. Dosya izinlerini kısıtlayın:
   chmod 600 .env

4. Üretim ortamında `.env` dosyası kullanmayın:
   - AWS: Systems Manager Parameter Store
   - GCP: Secret Manager
   - Heroku: Config Vars
   - GitHub Actions: Secrets

5. Hassas bilgilerin git geçmişine girmesini önlemek için:
   pip install detect-secrets
   detect-secrets scan > .secrets.baseline
   git commit .secrets.baseline
