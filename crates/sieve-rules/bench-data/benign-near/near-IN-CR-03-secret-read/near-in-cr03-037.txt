GCP legacy credentials and migration

Older GCP documentation recommended downloading a service account key JSON
and pointing GOOGLE_APPLICATION_CREDENTIALS to it:

    export GOOGLE_APPLICATION_CREDENTIALS="/path/to/service-account.json"

This pattern is being deprecated in favor of:
1. ADC for local development (gcloud auth application-default login)
2. Workload Identity Federation for CI/CD and external workloads
3. Attached service accounts for GCP-native workloads

If you have existing key files:
- Rotate and delete unused keys in IAM console
- Replace with ADC or WIF in your workflows
- Enable org policy "constraints/iam.disableServiceAccountKeyCreation"
  to prevent new key files from being created

Key files that do exist should have 0600 permissions and should NOT be
committed to version control or shared via chat/email.
