Using `~/.ssh/authorized_keys` on a server

This file controls which clients can log in via public key authentication.
Each line is one public key in the format:

    ssh-ed25519 AAAAC3NzaC1lZDI1NTE5AAAA... user@workstation
    # optional options prefix:
    from="192.168.1.*",no-port-forwarding ssh-rsa AAAAB3Nza... restricted

Common options:
  - `from="pattern"` — restrict to specific source IPs
  - `no-port-forwarding` — disable tunneling
  - `command="..."` — force a specific command (useful for deploy keys)

The file must be owned by the user, with permissions 600 or 644. Lines
starting with `#` are comments. The corresponding private key stays only
on the client machine and is never transmitted.
