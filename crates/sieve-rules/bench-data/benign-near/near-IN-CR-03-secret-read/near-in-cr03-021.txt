The `~/.aws/credentials` file stores AWS access keys on your local machine.
Its structure is INI-format with named profiles:

    [default]
    aws_access_key_id = AKIAIOSFODNN7EXAMPLE
    aws_secret_access_key = wJalrXUtnFEMI/K7MDENG/bPxRfiCYEXAMPLEKEY

    [production]
    aws_access_key_id = AKIAI44QH8DHBEXAMPLE
    aws_secret_access_key = je7MtGbClwBF/2Tk/h3yCo8nvbEXAMPLEKEY

Note: the keys shown above are AWS documentation example keys — they are
intentionally non-functional. The format helps you understand the structure.

Set file permissions to 600:  chmod 600 ~/.aws/credentials

Prefer IAM roles over long-lived access keys wherever possible — roles
issue temporary credentials automatically and don't need this file at all.
