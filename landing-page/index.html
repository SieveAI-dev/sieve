核心场景

四类检测，覆盖关键路径

出站脱敏 / 入站防篡改 / 危险工具拦截 / 签名审查——每一类都针对 crypto 开发者真实攻击面设计。

OUT-01

出站脱敏

自动擦除发往 LLM 的 prompt 中的私钥和助记词（BIP39 SHA-256 checksum 验证），5 秒状态栏通知，不打断工作流。

自动处理 · 无感知
IN-CR-02

入站地址替换检测

strsim Levenshtein 算法检测 LLM 返回内容中的近似 EVM / Solana 地址，GUI 弹出 60 秒倒计时强制确认。

Critical · GUI 确认
IN-CR-03

危险 tool_use 拦截

sieve-hook PreToolUse 拦截高风险工具调用（shell / 文件系统 / 签名路径）。终端 y/N 提示 + fail-closed，YOLO mode 不可关。

Critical · fail-closed
IN-CR-01

签名调用审查

EIP-712 typed data 解析展示人类可读摘要，GUI 120 秒强制确认窗口，超时自动拒绝。任何版本均不可跳过。

Critical · 120s 超时

透明性 / 信任

你能验证的，不只是声明

Sieve 用同一套标准审视自己。四个维度可以独立验证，不依赖我们的话。

🔏

sigstore 签名

每个 release 用 cosign 签名，密钥材料写入 Rekor 透明日志，可追溯。

cosign verify \ --certificate-identity \ doskey@TODO \ ghcr.io/doskey/sieve:latest
🔁

可复现构建

给定相同源码 + 环境，任何人都能得到 bit-for-bit 相同的二进制。

SOURCE_DATE_EPOCH=$( git log -1 --format=%ct ) cargo build \ --release --locked
🔌

零云依赖 verifier

所有检测逻辑本地执行，网络出口仅 LLM 上游请求。没有遥测，没有回调 home。

# 验证：无额外出站连接 # network egress = LLM upstream only # wireshark / mitmproxy 均可验证
📋
